# v6.3.3-velvet — April 2026

## Radio: smarter "found in library" matching + clickable badge

### Improved track matching

The library-detection logic for radio now-playing metadata has been completely
reworked.  Previously, each half of the ICY `"ARTIST - TITLE"` string was
searched independently, so any track by the same artist would light up the badge
even when the title was completely different.

The new strategy builds a **combined** query — `primaryArtist + bareTitle` — so
FTS5 must match both sides, eliminating false positives like:

- `JAMES HYPE & TITA LAU - DISCONNECTED` now correctly matches
  `James Hype - Disconnected (Remix)` (and all other versions), but will **not**
  match a random James Hype track with a different title.

Candidate chain (in priority order):

1. `primaryArtist(left) + bareTitle(right)` — normal `ARTIST - TITLE` order
2. `primaryArtist(right) + bareTitle(left)` — reversed-order stations
3. `bareTitle(right)` alone — title-only fallback when artist isn't in DB metadata
4. `bareTitle(left)` alone — reversed fallback

Primary artist extraction strips featured artists (`& / feat / ft / vs / x`).
Bare title extraction strips all parenthetical and bracketed suffixes
(`(Remix)`, `[Radio Edit]`, etc.) so all versions of a track are found.

### Clickable badge

The database-badge icon (cylinder icon, shown when a radio track is found in
your library) is now **clickable**:

- Clicking it navigates to the Search view pre-filled with the matched query
  (e.g. `"james hype disconnected"`) and immediately runs the search
- The badge tooltip now reads "Found in your library — click to search"
- Cursor changes to pointer with a subtle hover brightness effect
- Click is properly isolated via `stopPropagation` — the rest of the player-left
  area still opens the Now Playing modal as before

### ffmpeg-bootstrap improvements (from v6.3.2 unreleased section)

- **SHA256 checksum verification**: downloads BtbN `checksums.sha256` and
  verifies the archive before extraction
- **Windows x64 support**: downloads and extracts via PowerShell
- **macOS support**: signed static binaries from `ffmpeg.martin-riedl.de`
  (Intel x86_64 and Apple Silicon arm64)
- **Daily update check**: persists verified checksum, auto-updates if BtbN
  publishes a new build; timer uses `unref()` to never block server shutdown
- **CI workflow**: `.github/workflows/test-ffmpeg-bootstrap.yml` tests download
  and execution on all four platforms on every velvet tag push
