# v5.16.39-velvet — April 2026

## fix: switch Docker base image from Alpine to Debian slim

### Problem
`node:24-alpine` uses musl libc. The BtbN static ffmpeg builds dynamically link against glibc's `libmvec.so.1` (SIMD vector math) and several other glibc-only symbols. musl cannot provide these — the binary failed to load with "Error loading shared library libmvec.so.1" and dozens of missing symbol errors.

The earlier v5.16.38 attempt (`libc6-compat`) only creates the ELF interpreter symlink `/lib64/ld-linux-x86-64.so.2` — it does not supply the missing glibc shared libraries. It was not sufficient.

This caused **500 errors on all ffmpeg-dependent features** in Docker:
- Waveform scrubber
- Transcoding
- Radio recording
- YouTube downloads

### Fix
Switched the base image from `node:24-alpine` to `node:24-slim` (Debian Bookworm), where glibc is native and the BtbN binaries work without any compatibility layer.

Also added `wget` and `xz-utils` to the Debian install step — these were pre-installed in Alpine's BusyBox but need to be explicitly installed on Debian slim.
