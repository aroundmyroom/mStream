# v6.0.1-velvet — April 2026

## Playlist Rename + Security Fix

### feat: Playlist rename
You can now rename any regular playlist directly from the sidebar without
having to delete and recreate it.

- ✎ rename button added to every playlist row (between Share ↗ and Delete 🗑)
- A modal pre-fills the current name with the text selected — just start typing
- Returns an error if the new name is already taken
- The sidebar refreshes automatically and navigates to the renamed playlist if
  it was currently open

### fix: Auth cookie TTL 30 days (was 5 years)
The session cookie max-age was `157784630000 ms` (~5 years). It is now
`30 days`, which reduces the exposure window if a token is ever leaked.
Users will need to log in again after 30 days of inactivity.

### docs: Album Library design document
Complete design document added at `docs/album-browser-design.md`:
- Filesystem analysis of the `/media/music/Albums` vault
- Four structural patterns documented with real examples
- Data model, API endpoint design, UI/UX specification
- Implementation plan for the upcoming Album Library view
