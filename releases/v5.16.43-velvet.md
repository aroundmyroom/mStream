# v5.16.43-velvet — April 2026

## Bug fix

### ffmpeg auto-download 404 on fresh container start

**Symptom:** Container log showed:
```
HTTP 404 while downloading ffmpeg from https://github.com/BtbN/FFmpeg-Builds/releases/download/autobuild-2026-04-02-13-13/ffmpeg-master-latest-linux64-gpl.tar.xz
FFmpeg init failed — transcoding disabled
```
Transcoding returned 500 for every request.

**Root cause:** The URL `releases/latest/download/ASSET` tells GitHub "resolve your latest release, then redirect to that release's asset". If GitHub has just published a brand-new autobuild that is still uploading its assets, the redirect lands on a timestamped URL whose file doesn't exist yet → 404.

**Fix:** Changed URL to `releases/download/latest/ASSET` — this downloads directly from BtbN's `latest` tag, which they maintain as a stable pointer and only update after assets are fully uploaded. No redirects through GitHub's "latest release" resolver.
