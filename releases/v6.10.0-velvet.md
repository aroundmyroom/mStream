# v6.10.0-velvet — April 2026 — Deep Tag Dive

`v6.10.0` is a major feature release introducing AcoustID audio fingerprinting, a full MusicBrainz Tag Workshop, and a comprehensive Subsonic API compatibility overhaul that enables full library sync with Symfonium (and other strict clients).

---

## New: AcoustID Audio Fingerprinting

A new background worker fingerprints every song in your library using `fpcalc` (Chromaprint) and submits the fingerprint to the AcoustID API to obtain MusicBrainz IDs.

- Processes at ~2 songs/second with persistent progress — safe to stop and resume at any time
- New DB columns: `acoustid_id`, `mbid`, `acoustid_score`, `acoustid_status`, `acoustid_ts`
- New songs added by the file scanner are automatically queued
- `fpcalc` static binary pre-baked into the Docker image for both `linux/amd64` and `linux/arm64`

**Admin panel** (External Services → AcoustID):
- Enable/disable toggle
- API key input (masked — key never returned to the client)
- Live progress bar with stats: total / matched / no match / errors / queued
- Start / Stop worker on demand

To get an API key: [acoustid.org/api-key](https://acoustid.org/api-key)

## New: Tag Workshop — MusicBrainz Enrichment & Tag Writing

Once songs have AcoustID MBIDs, the Tag Workshop fetches full metadata from MusicBrainz and lets you review and apply it to your files.

### Enrichment worker

- Queries `musicbrainz.org/ws/2/recording/{mbid}` for every matched song (~1 song/second, respecting API rate limits)
- Picks the best release: prefers **Album** over **Single**, then earliest release date
- 7 new DB columns: `mb_album`, `mb_year`, `mb_track`, `mb_release_id`, `mb_enrichment_status`, `mb_enriched_ts`, `tag_status`

### Review UI

- **Album grid** with cover art, filter by: All / Missing tags / Year differs / Artist differs
- **Sort by:** Most differences / Most tracks / A–Z
- **Review modal:** side-by-side table of current file tags vs MusicBrainz data — mismatches highlighted orange, matches green
- Override artist/album name before accepting
- **Accept** writes tags to disk via ffmpeg using atomic temp-file write (never corrupts originals)
- **Skip** marks releases as reviewed without writing

### Batch operations

- Multi-select album cards with checkboxes
- "Select all on page" checkbox in toolbar
- **Accept Selected** processes a whole batch with per-track progress bar
- **Bulk accept casing** — one-click accept for releases where the only difference is letter casing

### Prerequisite banners

- Orange: no AcoustID fingerprints yet — links to AcoustID settings
- Red: fingerprints done but zero MB matches — prompts API key check
- Green: enrichment complete

### Supported formats for tag writing

mp3, flac, ogg, opus, m4a/aac, wav, wma, aiff

## Fix: Subsonic API — Full Library Sync Now Works (Symfonium & strict clients)

Several bugs were preventing full library sync in Symfonium and other strict Subsonic clients. All have been resolved.

### Bugs fixed

**Empty query parsing** — Symfonium sends `query=""` (with literal quote chars). These are now stripped before processing, so the empty-query path correctly triggers library enumeration.

**Zero count parameters ignored** — `songCount=0`, `artistCount=0`, `albumCount=0` were evaluated with `||`, falling through to a default of 20. Fixed to use `??` (nullish coalescing) so zero is respected and no unwanted extra results are returned alongside album pages.

**Non-standard response fields** — `searchResult3` was returning non-spec fields (`albumCount: 500`, `artistOffset`, etc.). Symfonium was reading `albumCount: 500` as "total albums = 500" and stopping pagination after one page thinking it had everything. These fields are now removed.

**Null optional fields crash strict parsers** — Symfonium's Moshi JSON parser uses `@NonNull` annotations on `coverArt` and `year`. Sending `"coverArt": null` or `"year": null` throws a hard exception, aborting the entire album import. All optional fields (`coverArt`, `year`, `artist`, `artistId`, `albumId`, `track`, `genre`, `duration`, `size`, `created`) are now omitted entirely when missing rather than sent as `null`. This is correct per the OpenSubsonic spec where these fields are optional.

**SQL LIMIT/OFFSET missing** — `getAllAlbumIds` and `getAllArtistIds` were loading the entire table into memory before slicing in JavaScript. They now pass `LIMIT ? OFFSET ?` directly to SQLite.

**Required fields missing** — `duration` and `created` were not always included on album objects. These are required by the OpenSubsonic AlbumID3 spec and are now always present.

**Child-vpath duplicate songs** — When a child vpath (e.g. an Albums subfolder) was used as the music folder, songs could appear twice — once under the child and once under the parent root. `resolveExcludePrefixes()` now correctly excludes parent-level rows when a child filter is active.
