# v6.6.1-velvet — April 2026

Improved "Add Channel" workflow in the Radio view: stations can now be discovered through a built-in radio-browser.info search before filling in the form manually.

---

## feat(radio): radio-browser.info station search

### Overview
Clicking **+ Add Channel** in the Radio view now opens a **Find a Station** search panel before showing the manual entry form. Users can search by name, country, and/or genre, instantly see results with logos, pick the best-quality stream, and have the add-station form pre-filled automatically.

### Search panel
- **Name / keyword** — free-text station name (e.g. "538", "3FM", "BBC Radio 2")
- **Country** — 2-letter code (`NL`, `US`) or full name (`Netherlands`, `Dutch`)
- **Genre / tag** — free-text (e.g. "pop", "dance", "jazz")
- Pressing **Enter** in any field triggers the search
- **Enter manually** skips the search and opens the blank form unchanged

### Quality selection
Results are deduplicated by station name. For each station the highest-quality stream is pre-selected automatically:
- Preference order: **MP3 > OGG > AAC**
- Within the same codec: highest bitrate wins (320 > 192 > 128 > 96 > 64 kbps)
- AAC is only selected when no MP3 or OGG stream exists

Alternative streams for the same station are hidden behind a **▾ N** toggle, each with a **Use** button to select that specific stream instead.

### Thumbnail column
Every search result shows the station's favicon/logo (40×40 px) to the left of its name — so it is immediately visible whether a channel has an icon or not. Broken or missing images fall back to the standard radio-wave placeholder.

### Pre-fill behaviour
Clicking **Select** (or **Use** on an alternative) pre-fills the add-station form:
| Field | Filled with |
|---|---|
| Name | Station name from radio-browser |
| Genre | Up to 3 tags from the station record |
| Country | Country field from the station record |
| Stream URL A | Best-quality resolved stream URL |
| Image URL | Favicon — **only if the URL ends in `.svg` or `.png`**; all other formats (ICO, WebP…) are ignored |

The filled form is fully editable before saving. Nothing is saved automatically.

### No backend changes
The search is performed directly from the browser to `de1.api.radio-browser.info` (public CORS-enabled API). No new server endpoints, no DB schema changes.
