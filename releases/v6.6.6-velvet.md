# v6.6.6-velvet — The Devil's Release — April 2026

Small but important fix: multi-disc album folders named `Cd 1`, `Cd 2` (mixed
case with a space) were not being recognised as disc sub-folders and appeared as
separate album entries instead of one unified album.

## fix: album CD-folder detection now handles mixed-case "Cd 1 / Cd 2" naming

The Album Library groups multi-disc sub-folders (e.g. `CD1`, `Disc 2`) into a
single album entry instead of showing each disc separately. This grouping relies
on a pattern match against the sub-folder name.

The detection regex matched `CD1`, `CD2`, `Disc 1`, `Disc 2` correctly, but
folders written as `Cd 1` or `Cd 2` (capital C, lowercase d, space before digit)
slipped through — the pattern required the digit to follow immediately or with
only an optional separator, but the mixed-case variant with a leading space was
not covered.

**Fix:** the `DISC_RE` pattern in `albums-browse.js` and the `_normaliseAlbumDir`
normaliser in `sqlite-backend.js` have been updated to match all common
capitalisation variants (`CD`, `Cd`, `cd`, `DISC`, `Disc`, `disc`, etc.) with or
without a space before the disc number.

Albums previously split into separate `Cd 1` / `Cd 2` entries will now correctly
appear as a single album with all tracks in one place.

> **Note:** if any such albums still display incorrectly, a library rescan will
> rebuild the grouping. A future update will verify broader discography detection
> coverage if needed.
