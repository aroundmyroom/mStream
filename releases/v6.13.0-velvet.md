# v6.13.0-velvet ŌĆö April 2026 ŌĆö The Tagged Albums

## Overview

v6.13.0 "The Tagged Albums" is a major quality-of-life release for users with well-organised libraries. The centrepiece is a complete **album edition / version tag system** ŌĆö the scanner can now detect and store edition information from ID3 tags, Vorbis comments, iTunes atoms, and APE tags across every audio format, with an intelligent heuristic fallback for files that have no edition tag at all.

The result is visible everywhere: version badges appear on album cards in the Library, Artist profiles, and Search. An **Edition filter bar** above the Album Library lets you drill into your Deluxe Editions or Hi-Res rips with a single click.

This release also ships significant improvements to the **AcoustID fingerprinting** worker (fewer false errors, smarter no-match classification) and the **MusicBrainz enrichment** worker (automatic HTTP 301 redirect following), plus the full **search completeness** fix from v6.12.10 is now accompanied by performance and accuracy improvements across the board.

## New: Album Version / Edition Tag System

### The problem

Users with multiple editions of the same album (original, remaster, deluxe, hi-res SACD) saw identical cards in the Album Library ŌĆö same name, same year. There was no way to tell them apart without opening each one.

### Scanner pipeline

The file scanner (`src/db/scanner.mjs`) gains a full multi-stage detection pipeline:

1. **`buildNativeMap()`** ŌĆö flattens `TXXX`, Vorbis, APE, and iTunes freeform atoms into a fast per-format lookup map, built once per file.
2. **`resolveTagField(field, songInfo, nativeMap)`** ŌĆö resolves one configured field name against all formats: handles `TXXX:KEY` prefix lookup, `music-metadata` cross-format `common.*` aliases, and raw Vorbis/APE/iTunes keys.
3. **`parseVersionHeuristic(string)`** ŌĆö a regex battery + Levenshtein fuzzy matcher for edition/quality keywords, gated on a bracket-or-keyword confidence check. Extracts canonical labels (`Deluxe Edition`, `Remaster`, `Hi-Res 24bit/96kHz`, `SACD`, `Live`, `Mono`, etc.) from album titles and folder names. Combined signals join with ` ┬Ę `.
4. **`deriveAlbumVersion()`** ŌĆö the main orchestrator: walks the configured field list in priority order, falls back to heuristic on album title, then folder name, then infers `Hi-Res Nbit/NkHz` from `bitsPerSample` / `sampleRate` when both indicate a high-resolution source.

### Supported tag formats

Every audio format is fully covered:

| Format | Where edition lives |
|---|---|
| MP3 | `TIT3` frame; `TXXX:EDITION`, `TXXX:VERSION`, etc. |
| FLAC / OGG / Opus | Vorbis comments: `SUBTITLE`, `DISCSUBTITLE`, `EDITION`, `VERSION`, ŌĆ” |
| M4A / AAC | iTunes freeform atoms: `----:com.apple.iTunes:EDITION`, etc. |
| WavPack / APE | APE tags: `Edition`, `Version`, `SubTitle`, ŌĆ” |

### New database columns

Three columns added to the `files` table (auto-migrated on first startup):

```sql
album_version        TEXT     -- e.g. "Deluxe Edition", "Hi-Res 24bit/96kHz"
album_version_source TEXT     -- "TIT3", "TXXX:EDITION", "heuristic:title", "inferred:audio", ŌĆ”
bit_depth            INTEGER  -- bitsPerSample from audio codec data
```

The `fts_files` FTS5 table was extended to include `album_version`, so searching for `"deluxe"` or `"24bit"` now also matches on the version field.

### Configurable tag field priority

The admin defines an **ordered list of tag fields** to try. The scanner walks the list and uses the first non-empty value it finds. Default order:

```
TIT3 ŌåÆ SUBTITLE ŌåÆ DISCSUBTITLE ŌåÆ TXXX:EDITION ŌåÆ TXXX:VERSION ŌåÆ
TXXX:ALBUMVERSION ŌåÆ TXXX:QUALITY ŌåÆ TXXX:REMASTER ŌåÆ TXXX:DESCRIPTION ŌåÆ
EDITION ŌåÆ VERSION ŌåÆ ALBUMVERSION ŌåÆ QUALITY ŌåÆ REMASTER
```

This list is stored in `save/conf/default.json` under `albumVersionTags` and is fully editable in the Admin UI.

### API changes

- `GET /api/v1/albums/browse` ŌĆö each album object now includes `album_version` and `bit_depth`.
- `POST /api/v1/db/metadata` ŌĆö response now includes `album-version` and `bit-depth`.
- All album query functions (`getArtistAlbums`, `getArtistAlbumsMulti`, `getAlbums`, `searchAlbumsByArtist`, search endpoints) propagate `album_version` through their SQL and API responses.
- New: `GET /api/v1/admin/db/album-version-inventory` ŌĆö returns a breakdown of files by `album_version_source`.
- New: `POST /api/v1/admin/db/params/album-version-tags` ŌĆö updates the priority field list in config.

## New: Album Library ŌĆö Version Badges & Edition Filter Pills

### Version badges

Version badges appear in four surfaces:

- **Album Library grid** ŌĆö badge on each album card beneath the title
- **Artist profile album grid** ŌĆö badge next to the year; sibling albums (same title, different versions) get a subtle outline highlight
- **Now Playing pane album grid** ŌĆö same badge treatment
- **Album songs view** ŌĆö a version header badge at the top of the track list

### Edition filter pills

When any album in the current view has a version tag, an **Edition** filter bar appears above the grid:

```
Edition:  [Deluxe Edition]  [Remaster]  [Hi-Res 24bit/96kHz]  ŌĆ”
```

**Opt-in behaviour**: pills start off ŌĆö all albums are visible. Clicking a pill filters to *only* albums with that version. Multiple pills can be active simultaneously (OR logic). Clicking an active pill deactivates it.

**Series filtering**: when version pills are active, album series (artist collections) are also filtered ŌĆö a series card is hidden if none of its member albums match any selected version.

**Filter state persistence**: active source filters and version pills survive navigation. Opening an album and pressing Back restores the exact scroll position and selected pills.

### Series view improvements

Opening a series (artist collection) from the Album Library now offers the full toolbar:
- **Version pills** pre-selected from the library view filter (can be adjusted independently inside the series)
- **Density controls** (List / Comfy / Compact) using the same `localStorage` preference as the main Library and Artist views
- **Version badges** on every album card inside the series

## Improvement: AcoustID Fingerprinting

### Short clip handling

Files shorter than 7 seconds are now classified as `not_found` immediately, without attempting fingerprinting. `fpcalc` cannot produce a reliable fingerprint for clips this short (jingles, DJ intro stabs, sound effects). This eliminates a class of spurious errors for libraries that include short promotional files.

### fpcalc failure handling

If `fpcalc` fails for any reason (corrupt audio, unsupported codec), the file is now marked `not_found` rather than `error`. These failures are permanent ŌĆö retrying every 7 days was wasteful. `not_found` correctly communicates that the file was processed but could not be identified.

### Error category simplified in Admin UI

The admin AcoustID stats card now shows a single **No match** row that combines `not_found` and `error` rows. Transient API errors self-resolve on retry and do not need a separate UI row to monitor.

## Improvement: MusicBrainz Enrichment ŌĆö HTTP 301 Redirect Following

MusicBrainz periodically merges duplicate recording entries. When a recording MBID is superseded, the MusicBrainz API returns `HTTP 301` with a `Location` header pointing to the canonical replacement MBID.

Previously the enrichment worker treated any non-200 response as an error, causing a permanent `MB HTTP 301` failure for affected files.

**Fix**: `mbLookup()` now follows 301/302/307/308 redirects, extracts the new MBID from the `Location` header, and retries automatically ŌĆö up to 3 hops. Files that previously accumulated `MB HTTP 301` errors in the Tag Workshop error list will now enrich correctly.

## Performance & Correctness (carried from v6.12.8ŌĆōv6.12.10)

> These fixes were shipped in earlier patch releases; this section summarises their cumulative effect.

- **FTS5 search** ŌĆö switched from `ORDER BY bm25()` to `ORDER BY rank`; FTS5 drives the join; `LIMIT 50` on rows (client already capped display). Result: searches that previously hung on large libraries now return in < 200 ms.
- **Search completeness** ŌĆö removed all `LIMIT` caps from `searchFiles`, `searchFolders`, `searchArtists`, and `searchArtistsNormalized`. All search tabs now show accurate, untruncated counts.
- **Album search** ŌĆö new `searchAlbumsByArtist()` uses `GROUP BY album` at the SQL level; art preservation via `MAX(aaFile)`.
- **Multi-disc album art** ŌĆö scanner falls back one directory level when the immediate folder has no cover art.
- **SQLite** ŌĆö 128 MB mmap, deferred ANALYZE on startup, statement cache for dynamic search SQL, page_size migration to 8192.
