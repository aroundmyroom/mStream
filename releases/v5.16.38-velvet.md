# v5.16.38-velvet — April 2026

## fix: ffmpeg "not found" on Alpine Docker

### Problem
BtbN static ffmpeg binaries compile their ELF interpreter path as `/lib64/ld-linux-x86-64.so.2`. Alpine Linux uses musl and does not ship this path, so the kernel returned `ENOENT` ("not found") when trying to execute ffmpeg — despite the binary being present and having correct permissions.

This affected **all ffmpeg-dependent features** in the Docker container:
- Waveform scrubber (500 errors on every track)
- Transcoding
- Radio recording
- YouTube downloads

### Fix
Added `apk add --no-cache libc6-compat` to the Dockerfile. This package creates `/lib64/ld-linux-x86-64.so.2` as a symlink to musl's own ELF loader. Since the BtbN binary is fully statically compiled (all glibc symbols baked in), musl's loader can start it without any further glibc runtime dependency.
