# v6.11.0-velvet — April 2026 — Cast to Speaker · Audio Format Display · Server Remote Redesign

Minor release with four headline features: cast playback to the server speaker via mpv, live audio format info (bitrate · kHz · Stereo) in the queue panel, a completely redesigned Server Remote page, and a full Home screen overhaul. Also ships the Server Audio admin rework, gapless/crossfade mutual exclusion, and several scan-quality fixes.

---

## New: Cast to Server Speaker (MPV Cast)

A new **Cast** button in the player controls lets you route playback from your browser directly to the server's speaker via mpv — turning any device into a remote control for the room.

- Toggle button in the player bar: click once to cast; click again to return audio to the browser.
- When casting, the browser audio element is muted and the current song is mirrored to mpv at the same playback position.
- Play/pause, next/prev, Auto-DJ, and queue all continue to work normally; mpv mirrors every track change.
- **Per-user opt-in permission** (`allow-mpv-cast`): admin must enable the Cast button per user in Admin → Users. Default: off.
- Requires Server Audio to be enabled and mpv running.
- New i18n keys: `player.ctrl.castToMpv`, `player.ctrl.castToBrowser`, `player.toast.castingToMpv`, `player.toast.castToBrowser`, `player.toast.castQueueEmpty`.

## New: Audio Format Info in Queue Panel

The now-playing card in the queue panel now shows the audio format of the current track — format, bitrate, sample rate and channel mode — drawn from the database.

- Displays as: **FLAC · 1027 kbps · 44.1 kHz · Stereo** (or MP3, WAV, etc.)
- Data is stored during scan in three new DB columns: `bitrate` (kbps integer), `sample_rate` (Hz integer), `channels` (integer).
- If a song enters the queue without tech-meta (e.g. from a restored localStorage queue), an async fetch to `/api/v1/db/metadata` fills it in and updates the display automatically.
- All API endpoints that return song metadata (`recently-played`, `most-played`, `recently-added`, `metadata`, `autocomplete`, `smart-playlists`, etc.) now include `bitrate`, `sample-rate`, and `channels` fields.

### Fix: Scanner now correctly extracts bitrate for FLAC and WAV files

The `_needsBitrate` scan path previously called `parseFile(path, { skipCovers: true })` without `duration: true`. For lossless formats (FLAC, WAV, AIFF), bitrate is not stored in the file header — it must be computed from filesize and duration, which requires music-metadata to read the full file.

**Fix:** the scanner now uses `{ skipCovers: true, duration: true }`, and a fallback calculation (`fileSize × 8 ÷ duration ÷ 1000`) is applied when music-metadata still returns null (e.g. for unusual containers).

## New: Server Remote Page — Complete Redesign

The Server Remote page (`/server-remote`) has been completely rebuilt with a fully functional mobile-optimised remote control UI.

- **Now-Playing bar**: album art, title, artist, audio format, and a live scrub bar.
- **Transport controls**: prev / play-pause / next, shuffle, repeat, volume.
- **Three tabs**: Queue / Browse / Search.
  - **Queue tab**: full queue list with current track highlighted, add/remove.
  - **Browse tab**: file-browser tree for navigating and queueing folders.
  - **Search tab**: live search with add-to-queue buttons.
- **Auto-DJ**: optional Auto-DJ toggle in the remote, with Last.fm similar-artists mode when an API key is configured.
- Dark/light theme toggle; responsive layout down to 390 px (iPhone SE).
- New `.np-tech` element shows format info matching the queue panel.

## New: Home Screen — Full Implementation

The Home view (default landing screen) now shows a rich personalised dashboard:

- **Hero greeting + stats strip**: time-aware greeting ("Good morning/afternoon/evening, [Name]") with today's song count, this-week count, and a listening-streak indicator (highlighted when > 1 day).
- **Continue Listening**: recently played songs deduplicated by album so you see one entry per album/artist to resume.
- **Recently Added**: songs most recently added to the library — "NEW" badge.
- **On This Day**: songs played on this same calendar day in previous years, powered by `play_events`.
- **Mood Quick-Picks**: most-played songs split into "Nostalgia" (> 10 years old) and "Current Favourites" (recent). Falls back to a plain Most Played shelf when the dataset is too small to split meaningfully.
- Shelves that have no data are hidden — the view is always clean.
- New `GET /api/v1/db/home-summary` endpoint: returns `todayCount`, `weekCount`, `streak`, and dated sections from `play_events`.

### Shortcuts screen

The former "Home" nav item is now **Shortcuts** (2×2 grid icon). It retains all its previous content: radio stations, podcasts, playlists, recently/most played, and "Because you listened to …" shelves. This frees the Home slot for the new personalised dashboard above.

## Improved: Server Audio Admin — Rework

The Admin → Server Audio card has been completely reworked to be self-documenting and actionable:

- **Status badge**: Running / Enabled (not started) / Disabled — immediately clear.
- **Auto-prepare ALSA output** toggle (`serverAudio.autoUnmute`): when enabled (default: on), mStream tries to unmute key ALSA mixer channels and set volume to 90% before starting mpv. Prevents silent playback on a freshly booted Linux server.
- **Audio health check** (`GET /api/v1/server-playback/audio-health`): verifies mpv binary, ALSA tools (`amixer`/`aplay`), mixer mute state, and detected sound cards. Returns a structured health report shown inline in the admin panel.
- **Backend audio fix** (`POST /api/v1/server-playback/audio-health/fix`): one-click unmute + volume set for `Master`, `Speaker`, `PCM`, `Headphone` ALSA controls.
- **Guided test**: single button that runs detect → check → fix → start in sequence and prints a pass/warn/fail report.
- Every action button shows a busy label while the request is in flight.
- Each control row includes a plain-English explanation of what it does.
- 44 new `admin.serverAudio.*` i18n keys, all 12 locales.

## Improved: Gapless / Crossfade — Mutual Exclusion with Restore

Gapless playback and crossfade are now properly mutually exclusive:

- **Enabling gapless** automatically sets crossfade to 0 and saves the previous crossfade value as `crossfadeRestore` in localStorage.
- **Disabling gapless** restores crossfade to the saved value (or 0 if none was stored).
- This prevents the silent-skip / double-trigger bug where having both a stored crossfade value and gapless enabled would cause neither transition path to fire correctly.
- The Gapless and Crossfade buttons in the player bar both stay in sync via `_syncGaplessButton()` and `_syncCrossfadeButton()`.

## New: Scan History — Last Scanned Timestamp

A new `scan_runs` table records the start and finish time of every completed scan. The Admin → Database stats card now shows a **Last Scanned** timestamp. The `recordCompletedScan()` function is called from the scanner's `finish-scan` endpoint.

## Fix: i18n — Dynamic Player Views Re-Render on Language Switch

Previously, only static `data-i18n` elements updated when the user switched language in Settings. Dynamic views rendered from `webapp/app.js` (Auto-DJ, Playback, File Explorer, playlists, smart-playlists, podcast episodes) kept stale translated text until navigated away and back.

Fixed: `I18N.onChange(...)` now calls `_rerenderCurrentViewForLanguageChange()` which re-enters the current active view, regenerating its HTML with the new locale immediately.

---

## Fix: Per-User Server-Remote Access Permission

`allowServerRemote` is now only `true` when both Server Audio is enabled in config AND `allow-server-remote` is not explicitly set to `false` for the user. Previously it defaulted to `true` even when mpv was not configured, causing a confusing broken page.
