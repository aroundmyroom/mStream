# v6.8.0-velvet — April 2026 — Artist Library overhaul + admin image moderation

`v6.8.0` delivers the full Artist Library rollout and the follow-up fixes that landed right after real-world testing: better normalization, corrected artist drill-down behavior, image moderation tools in admin, safer background hydration, and clearer progress visibility for large libraries.

---

## Artist Library (new)

### Home + browse flows

- New **Artists** entry in the player navigation.
- Home shelves:
	- **Most Songs in Library**
	- **Most Played Artists**
	- **Recently Listened**
- Shelf drag-reorder with per-user persistence.
- A–Z and `0-9` pill browse.
- Debounced full-text artist search.

### Per-artist navigation and album discovery

- Album grid built from canonical artist groups + raw variants.
- Series grouping for nested structures.
- Duplicate-name fallback (folder-name display when many albums share one tag).
- Correct breadcrumb/title progression and level-aware back-navigation.
- Fixed letter-filter UX so selecting a letter does not flash HOME shelves first.

### Smart normalization and index rebuild

- Expanded artist cleanup pass for common corrupt formats:
	- underscore-prefixed numbers (`01_Artist`)
	- bare track-number artists (`01`, `42`) excluded from artist index
	- parenthetical remix-only artist tags
	- malformed remix suffix variants via similarity merge heuristics
- Added admin trigger: `POST /api/v1/admin/artists/rebuild-index`.

---

## Artist images: moderation + repair workflow (new)

### Player-side marking (admin)

- Admins can mark an artist image as wrong directly from artist cards/lists.
- Wrong-image action is compact and non-intrusive in the card UI.

### Admin-side repair panel

- New **Admin → Artists** view with:
	- **Missing** queue
	- **Wrong** queue
	- Discogs candidate picker
	- direct custom URL apply
	- clear wrong-flag action

### New endpoints

- `POST /api/v1/artists/mark-image-wrong`
- `GET /api/v1/admin/artists/image-audit?kind=missing|wrong`
- `GET /api/v1/admin/artists/discogs-candidates?artistKey=...`
- `POST /api/v1/admin/artists/apply-image`
- `GET /api/v1/admin/artists/hydration-status`

### Data model updates

- `artists_normalized.image_flag_wrong` added and preserved across index rebuilds.
- Wrong flag auto-clears when a new image is applied.

---

## Performance and anti-hammer safeguards

Large libraries (10k+ missing artist images) now run with explicit back-pressure:

- Background queue cap (`800`) with dropped-item tracking.
- Adaptive hydration pacing:
	- success: `~1.4 s`
	- no-image/skip: `~2.2 s`
	- error: `~4.0 s`
- Failed/no-image attempts mark `last_fetched` so the same artists are not hammered repeatedly.
- Hydration status telemetry exposed to admin (running state, queue depth, counters, Discogs readiness).

## Discogs and source behavior

- Discogs-first artist-image lookup, Last.fm fallback.
- Last.fm known placeholder/no-image URLs are rejected.
- Admin UI clearly indicates when Discogs is unavailable (disabled or missing API credentials).
