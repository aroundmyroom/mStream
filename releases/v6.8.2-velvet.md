# v6.8.2-velvet — April 2026 — Artist folder controls + image review workflow

`v6.8.2` focuses on Artist Library control and moderation quality: per-folder Artist include toggles, safer root/child filtering logic, and a stronger Admin image-review flow.

---

## New: per-folder Artist include toggle (`artistsOn`)

Admin → Directories now has an **Artists: On/Off** toggle (default ON) for all non-excluded folders.

When turned off for a folder, that folder is excluded from Artist Library data:

- Artist Home shelves
- A-Z browse and search
- artist image hydration/missing queues

Changes apply immediately by rebuilding `artists_normalized` and invalidating Artist Home cache.

---

## Fix: root/child folder filtering regression

Artist indexing uses ROOT DB vpaths with child folder prefixes. A direct vpath-only filter could empty Artist Home in mixed root/child toggle setups.

`rebuildArtistIndex()` now applies artistsOn with parent-vpath filepath-prefix include/exclude rules, matching the existing child-vpath architecture.

Result:

- Root OFF + selected child ON works correctly
- Root ON + selected child OFF excludes only that subtree

---

## Admin Artists improvements

### With-image review list

Admin → Artists now has a **With image** filter so existing portraits can be reviewed directly.

Per selected artist:

- **Yes: image is OK**
- **No: mark wrong**

This makes quality review possible without waiting for items to appear in the wrong queue first.

### Manual URL preview

Applying a custom image URL now includes a live preview and invalid-image warning before apply.

---

## Artist normalization hardening

Added normalization for vinyl tracklist-prefix corruption in artist tags:

- `A1 Artist`, `A1. Artist`, `A1, Artist`, `A1.Artist`
- nested malformed prefixes like `02 B1.Tom Hooker`

These variants now merge into canonical artists instead of polluting the Artist Library with pseudo-artists.
