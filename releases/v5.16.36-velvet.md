# v5.16.36-velvet — April 2026

## Bug fixes

### ffmpeg auto-download fails on Docker (Alpine BusyBox tar)

The ffmpeg bootstrap used `--wildcards '*/bin/ffmpeg'` which is a GNU tar extension.
Alpine Linux ships BusyBox tar which does not support `--wildcards` and exits with
code 1, producing the error:

```
[ffmpeg-bootstrap] Failed to download ffmpeg: tar extraction failed (code 1): ...
--exclude PATTERN   Glob pattern to exclude
```

**Fix:** Replaced wildcard paths with explicit paths derived from the archive name
(e.g. `ffmpeg-master-latest-linux64-gpl/bin/ffmpeg`). This works with both
BusyBox tar (Alpine/Docker) and GNU tar (bare-metal Linux).

### Bootstrap config write silently failed — server started with no folders

If `bootstrapFromEnv()` threw an exception for any reason (e.g. a permission error
before `fsp.writeFile`), the exception propagated uncaught. The server continued
(Node's top-level await behaviour), but `config.setup()` then ran and created a
fresh empty config file with only `{"secret": "..."}` — silently discarding the
intended folder configuration.

**Fix:** Wrapped `bootstrapFromEnv()` in a `try/catch`. Errors are now printed as
`[bootstrap] Failed to write initial config: <reason>` and the server continues.
The failure is visible in `docker logs mstream-velvet` so users can diagnose it.

## How to update

```bash
docker compose pull
docker compose down
docker compose up -d
```

If you had a `save/conf/default.json` that only contains `{"secret": "..."}`,
delete it before restarting — the bootstrap will recreate it correctly on the
next start.
