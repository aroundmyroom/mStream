# v6.7.3-velvet — Smarter album art search — April 2026

Art searches (Discogs, Deezer, iTunes) now automatically derive the artist name
from the folder structure when no artist tag is available, giving far better
results for albums organised as `Artist / Album` on disk.

---

## fix: art search uses parent folder as artist when artist tag is absent

Albums stored under an artist sub-folder (e.g. `Albums/Isaac Hayes/Branded`)
previously searched only for the album title ("Branded"), returning unrelated or
zero results on all three services. The search now extracts "Isaac Hayes" from the
parent folder and includes it in every query.

### Where this applies
- **Album Library art picker** — Discogs, Deezer, and iTunes searches from the
  album detail view all use the derived artist.
- **Now Playing modal art picker** — Deezer and iTunes button searches fall back
  to the folder-derived artist when the song's artist tag is empty (Discogs
  already had a server-side filepath parser; this brings Deezer and iTunes to the
  same level).
- **Server-side `album.artist` field** — `albums-browse.js` now also populates
  `album.artist` from the parent folder when the album folder name itself does
  not contain the `Artist - Title` convention. This improves displayed artist
  metadata in the detail header and in queue entries.

### Logic
- `Albums/Isaac Hayes/Branded` → artist `"Isaac Hayes"`
- `Albums/Isaac Hayes - Branded` → artist `"Isaac Hayes"` (existing behaviour)
- Single-character index folders (`A`, `B`, …) are skipped
- Paths without a meaningful parent (fewer than 3 segments) are ignored
