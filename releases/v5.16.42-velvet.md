# v5.16.42-velvet — April 2026

## Bug fix

### Transcoding route — `url.charAt is not a function` crash

All transcoding requests were failing with a 500 error:

```
TypeError: url.charAt is not a function
    at Module.getVPathInfo (src/util/vpath.js:8:11)
    at src/api/transcode.js:114:28
```

**Root cause:** Express 5 uses path-to-regexp v8, which changed how wildcard route parameters work. The `{*filepath}` wildcard no longer returns a plain string in `req.params.filepath` — causing `getVPathInfo()` to receive a non-string value and throw immediately.

**Fix:** Read the file path from `req.path` (always a plain decoded string in Express) and strip the `/transcode/` prefix, then `decodeURI()` before passing to `getVPathInfo()`.
