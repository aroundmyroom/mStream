# v5.16.31-velvet — March 2026

## Admin UI

- **Sidebar restructured** — Settings, Database, Backup, Transcoding, Federation, Logs, Scan Errors and Lock Admin API moved under the **Config** section; About moved to the bottom under its own **Server** section
- **Credits updated** — About view now correctly attributes mStream to Paul Sori and mStream Velvet features/maintenance to Dennis Slagers (AroundMyRoom); GitHub and Discord links added
- **Logout standardised** — "Sign Out" renamed to "Logout" everywhere (player footer, admin sidebar, confirm dialog) to be consistent

## Bug fixes — settings persistence

- **Home layout not syncing across devices** — `ms2_home_order` and `ms2_home_hidden` keys used `S.user` (undefined) instead of `S.username`, causing all users to share a single `undefined` slot and server-synced layout to be written to a key never read back. Fixed to `S.username`.
- **Home prefs missing from bulk sync** — `home_order` and `home_hidden` were not included in `_collectPrefs()`, so the login-time settings push never carried them to the DB. Fixed.
- **Mute state not persisted** — clicking Mute did not write to localStorage or sync to DB. Mute state (`ms2_mute`) is now saved and restored on login, including the pre-mute volume so un-muting returns to the correct level.
- **Shuffle state not persisted** — `S.shuffle` was toggled in memory only, lost on page reload. Shuffle state (`ms2_shuffle`) is now saved and restored on login.

## Docker — GitHub Container Registry

- New workflow `.github/workflows/docker-publish.yml` automatically builds and pushes a multi-arch Docker image (`linux/amd64` + `linux/arm64`) to `ghcr.io/aroundmyroom/mstream` on every `v*-velvet` tag — no manual steps needed
- `compose.yaml` updated to reference `ghcr.io/aroundmyroom/mstream:latest` by default
- `docs/docker.md` updated with pull instructions

```shell
docker pull ghcr.io/aroundmyroom/mstream:latest
```
