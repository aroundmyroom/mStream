# v5.16.34-velvet — April 2026

## What's new

### feat: Docker first-run auto-config via environment variables

Fresh Docker deployments can now be configured entirely from `compose.yaml` without touching any file inside the container.

Set `MSTREAM_MUSIC_DIR` to the path where your music volume is mounted and mStream will automatically write `save/conf/default.json` on the very first start. On every subsequent restart the env vars are ignored — the config already exists.

**Environment variables are completely optional.** The server works perfectly without them; edit `save/conf/default.json` directly (or use the admin UI) for full control.

**When env vars are NOT sufficient — edit the config directly instead:**
- Multiple mount points / multiple music volumes
- Child-vpaths (e.g. an `Albums` vpath with `albumsOnly: true` inside a parent vpath)
- `albumsOnly` / `filepathPrefix` / `includeFilepathPrefixes` filtering rules
- Any advanced folder layout that does not fit the flat single-root pattern

**Available variables:**

| Variable | Default | Description |
|---|---|---|
| `MSTREAM_MUSIC_DIR` | _(empty)_ | Path to music volume inside container. Only variable that triggers bootstrap. |
| `MSTREAM_ADMIN_USER` | _(empty)_ | Admin username. If omitted: open mode, no login required. |
| `MSTREAM_ADMIN_PASS` | _(empty)_ | Admin password. Must set both USER and PASS together. |
| `MSTREAM_ENABLE_AUDIOBOOKS` | _(empty)_ | Set to `"true"` to create an AudioBooks/Podcasts vpath. |
| `MSTREAM_ENABLE_RECORDINGS` | _(empty)_ | Set to `"true"` to create a Recordings vpath and enable radio. |
| `MSTREAM_ENABLE_YOUTUBE` | _(empty)_ | Set to `"true"` to create a YouTube downloads vpath. |
| `MSTREAM_AUDIOBOOKS_SUBDIR` | `Audiobooks` | Sub-folder name inside music dir for AudioBooks. |
| `MSTREAM_RECORDINGS_SUBDIR` | `Recordings` | Sub-folder name inside music dir for Recordings. |
| `MSTREAM_YOUTUBE_SUBDIR` | `YouTube` | Sub-folder name inside music dir for YouTube downloads. |

See `compose.yaml` for a fully annotated example, and `docs/docker.md` for the complete setup guide.

## Bug fixes

### fix: album browsing — songs from wrong vpath when clicking an album

Clicking an album in the Albums view was loading songs from **all** vpaths — including 12-inch, vinyl, and other non-album folders — because `viewAlbumSongs` sent no vpath/prefix filter. For users with strict child-vpath layouts (e.g. albums-only folder), playback would silently include tracks that had no business appearing there.

Fixed across three layers: the Vue method now calls `_albumsOnlyFilter()`, the API endpoint forwards the prefix filter, and the SQL query enforces it via `includePrefixClauses`.

### fix: duplicate albums — multiple pressings collapsed into one entry

`getAlbums`, `getArtistAlbums`, and `getArtistAlbumsMulti` deduplicated by `album + year`, so two releases with the same title and year (e.g. a Japan press and an EU remaster from the same year) appeared as a single album.

Fixed: deduplication now uses `album + physical directory`. Each unique folder becomes its own entry. Multi-disc folders (`CD1`, `CD2`, `Disc 1`, etc.) are still collapsed into their parent album via `_normaliseAlbumDir()`.

### fix: admin panel theme switcher clipped when sidebar has a scrollbar

On screens where the sidebar is shorter than its content, the three theme pills (Velvet / Dark / Light) were sliced horizontally through the middle and partially invisible.

Root cause: `#sidenav` is `flex-direction: column`; without `flex-shrink: 0`, flex was shrinking the pill container's height; `overflow: hidden` then clipped the buttons. Fixed with a single CSS rule.
