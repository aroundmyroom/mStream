# v6.3.2-velvet — Mea Culpa Release (April 2026)

This release contains only CI/CD and git hygiene fixes. No new features or bug fixes in the application itself.

## What changed

### Security
- Scrubbed `save/db/default.json` from entire git history using `git filter-repo` — the file contained secrets (tokens, passwords) that had been accidentally committed
- Added `save/db/*.json` and `save/db/*.sqlite` to `.gitignore` to prevent future occurrences

### CI/CD
- Fixed Docker build failure: Trivy SARIF upload was failing with "only one run per tool/category" error — added unique `category: trivy-fs` and `category: trivy-image` to the two upload steps
- Disabled Electron build workflow — velvet is a server-only fork, Electron builds are not published
- Fixed CodeQL upload action version (`upload-sarif@v4`)

### Repository
- Rebased `master` onto `upstream/sqlite` baseline — fork is now 0 commits behind the correct upstream base (IrosTheBeggar/mStream `sqlite` branch)
- Removed all stale remote branches (old PR branches, dependabot branches, etc.)
