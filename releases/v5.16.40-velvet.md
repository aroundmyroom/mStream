# v5.16.40-velvet — April 2026

## fix: yt-dlp EACCES on Docker — chmod after download and on startup

### Problem
`_ensureYtdlp()` used `fsp.access()` (existence check only) to decide whether the binary was ready. If the file existed but lacked execute permission — which can happen after a fresh container pull where the previous run wrote the file without `+x` — the function returned the path without fixing permissions, causing every yt-dlp call to fail with `spawn EACCES`.

### Fix
- On startup: check `fs.constants.X_OK`; if the file exists but isn't executable, `chmod 0o755` it immediately and return — no re-download needed
- After fresh download: always `chmod 0o755` the newly downloaded binary
