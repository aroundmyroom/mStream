# v6.12.7-velvet — April 2026 — Playing Now, Artist Images & DLNA Polish

## Highlights

- **Playing Now view** — completely redesigned with hero background, artist image, bio, similar-artist chips and album grid
- **TheAudioDB** added as image source for artist photos (runs between Discogs and Last.fm fallback)
- **No more head-cropping** of artist portrait photos — sharp now crops from the top
- **CUE chapter navigation** for every file type (not just audiobooks)
- **AcoustID regression fixed** — processed-count no longer resets on every server restart
- **DLNA**: MP3 passthrough, audiobook `upnp:class`, invalid-XML sanitiser, HEAD probe support
- **Security**: timing-safe Subsonic password comparison; full IPv6 SSRF hardening for podcasts/radio
- **Branding**: "mStream Velvet" consistently used in all user-visible strings

## Playing Now view redesign

The Playing Now view (`/playing-now`) is fully rebuilt:

- **Hero section** — blurred album-art background, large album art thumbnail, song title / artist / album overlay
- **Artist image** — fetched from the local image cache via the new `GET /api/v1/artists/image?artist=` endpoint; triggers background hydration if the artist is not yet cached
- **Bio** — loaded from Last.fm via the new `GET /api/v1/lastfm/artist-info` endpoint; collapsible "Read more" button for long bios
- **Similar artists** — chip list from Last.fm; chips are highlighted when the artist is in your library; clicking navigates to the artist profile
- **Album grid** — library albums by the current artist displayed below the bio
- **Smooth fade transition** on song change

## Artist image improvements

- **TheAudioDB** is now tried as a second source (after Discogs, before Last.fm). It covers many artists missed by Discogs.
- **`position: 'top'` in sharp resize** — portrait photos no longer crop off the head. All newly fetched images use top-anchored cropping. `object-position: top` CSS applied to both the Playing Now image and the artist profile image.
- **Hydration queue limit** raised from 800 → 2000; seed endpoint limit raised to 9999.
- **New admin endpoint**: `POST /api/v1/admin/artists/hydrate-tadb-noimage` — queue a TheAudioDB-only retry pass for artists that previously returned no image.
- **New admin endpoint**: `GET /api/v1/admin/artists/tadb-candidates?artistKey=` — preview TheAudioDB candidate images for an artist before applying.
- **New DB function**: `getArtistRowByName(name)` — resolves a raw ID3 tag value (including featuring variants) to a canonical artist row.
- **`downloadJson` hardened** — configurable `timeoutMs`, `connection: close` header, and socket drain on non-200 responses to prevent open handle leaks.

## Artist profile page

- **Similar Artists section** now appears above the album library (was below).
- Artist image uses `object-position: top` — portraits stay head-visible.

## CUE sheet chapter navigation for all files

- **On-demand CUE extraction**: files with `cuepoints IS NULL` (newly added or sidecar `.cue` added after scan) now have their CUE data extracted on-the-fly at first play. Supports embedded `CUESHEET` tag (FLAC etc.) and sidecar `.cue` files. Result stored to DB with `[]` sentinel to avoid re-probing empty files.
- **Chapter nav buttons activate for any file** with ≥ 2 cuepoints — not only audiobook-vpath files. Speed control and book-position memory remain audiobook-only.

## AcoustID fix — processed count no longer resets

**Bug**: AcoustID can return a fingerprint match (score ≥ 0.50) but with an empty `recordings[]` array — fingerprint recognised but no MusicBrainz recording linked yet. The worker was writing `status='found'` with `mbid=null`. The startup migration (which resets `found+mbid=null` rows back to unprocessed) was then re-queuing ~4 000 files on every restart.

**Fix**: Worker now writes `status='not_found'` when `mbid` is null. The startup migration is now a true no-op.

## MusicBrainz enrichment worker

- **DB write retry loop** — retries up to 3 minutes on `SQLITE_BUSY` (AcoustID worker holds the write lock during batch runs).
- WAL mode + `busy_timeout=60 000 ms` applied at startup.
- Pending rows left from an interrupted previous run are reset at startup.

## Last.fm — artist resolution for Auto-DJ

- `resolveArtistNamesForDJ()` normalises `&` ↔ `and` before matching Last.fm artist names against the library, so "AC/DC" from Last.fm correctly matches "AC/DC" in your collection.
- New `GET /api/v1/lastfm/artist-info` endpoint returns bio (HTML stripped), genre tags, listener/play counts.

## Subsonic — timing-safe password comparison

Subsonic password checks now use `crypto.timingSafeEqual` to prevent timing-based oracle attacks on token and plaintext password paths.

## DLNA polish

- **Invalid XML sanitiser** — strips XML 1.0 control characters and UTF-16 surrogate halves from DIDL metadata. Mojibake-ridden ID3 tags no longer crash strict DIDL parsers on Sonos, Plex, or LG TVs.
- **Audiobook `upnp:class`** — tracks in an `audio-books` vpath are advertised as `object.item.audioItem.audioBook`; library containers as `object.container.album.audioBook`.
- **MP3 passthrough** — MP3 inputs use `-c:a copy` instead of re-encoding to 192 k MP3, saving CPU.
- **HEAD probe support** — Sony and Marantz renderers send a HEAD + `TimeSeekRange` probe before GET. Returns 200 immediately.

## SSRF hardening (podcasts & radio)

The private-range blocklist is now comprehensive:

- IPv6 loopback (`::1`, `::`), ULA (`fc00::/7`), link-local (`fe80::/10`)
- IPv4-mapped IPv6 (`::ffff:a.b.c.d`) — recursively checks the v4 portion
- APIPA / cloud metadata (`169.254.x.x`)

## Artist Library — density controls

Three view modes for the Artist Library, toggled by buttons in the toolbar, persisted per user in `localStorage`:
- **List** — single-column rows (default)
- **Comfy** — 160 px card grid
- **Compact** — 100 px card grid

## FTS5 search fix

Artist and folder search queries are now wrapped in double-quotes to form literal phrases, preventing FTS5 syntax errors on names that contain `.`, `(`, `)`, `*`, or `-`. Queries shorter than 3 characters are rejected early (trigram index minimum).

## Child-vpath access control fix

Users who do not have access to a child vpath (e.g. a user with "Music" but not "12-inches") now have the child sub-folder automatically excluded from all DB queries via `computeChildExclusions()`.

## Scanner — "Recently Added" timestamp fix

`_isReindex` is now only set for stale (already-indexed) files. Newly added files receive a wall-clock `ts` so they appear correctly at the top of Recently Added.

## Branding

"mStream Velvet" consistently used across all 12 locale files, HTML titles, admin UI, and user-visible strings (was "mStream" in many places).
