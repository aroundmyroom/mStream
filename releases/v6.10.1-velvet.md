# v6.10.1-velvet ŌĆö April 2026 ŌĆö Gapless & Crossfade fixes, Subsonic scrobbling

Patch release with several audio playback fixes and a new Subsonic scrobble-forwarding feature.

## Fix: Gapless playback fully restored

Gapless playback had been silently broken since v6.7.5 due to two separate issues:

**Issue 1 ŌĆö blup/click on every gapless transition:**  
The v6.7.5 crossfade volume-spike fix removed a hard `gain.value = 1` set in `_doXfadeHandoff`. That fix was correct for crossfade (a `setValueCurveAtTime` is still running on the audio thread at handoff time), but it regressed gapless: the incoming gain node was left at the tail of a 20ms micro-ramp instead of being snapped to 1.0 when `ended` fired, causing an audible blup on every track transition.

**Issue 2 ŌĆö both gapless and crossfade blocks could silently skip:**  
When gapless was enabled and a non-zero crossfade value was also stored, both `timeupdate` trigger blocks skipped ŌĆö the crossfade block lacked a `!S.gapless` guard, and the gapless block had a now-redundant `S.crossfade === 0` guard. Result: normal `Player.next()` fired with a gap.

**Fix:** `_doXfadeHandoff` now applies the v6.7.0 hard gain snap only when `S.gapless` is true; crossfade path is unchanged. The two `timeupdate` guards (`!S.gapless` on crossfade, none on gapless) make them mutually exclusive.

---

## Fix: Crossfade could fire while gapless was active

Added `!S.gapless` guard to the crossfade trigger in `_onAudioTimeupdateUI`. Previously if a user had gapless enabled and any non-zero crossfade value stored from prior use, the crossfade path would win and override the gapless transition.

## New: Subsonic scrobble forwarding (Last.fm + ListenBrainz)

Subsonic-compatible apps (Symfonium, DSub, Ultrasonic, Tempo ŌĆ”) can now forward their scrobbles to your linked Last.fm and/or ListenBrainz accounts.

- **Per-user opt-in** ŌĆö each user independently enables forwarding per service
- **play_events** ŌĆö Subsonic submissions now insert into `play_events` (source=`subsonic`), so plays from external apps count in Your Stats, Home screen history, and Wrapped
- **Subsonic settings page** ŌĆö a new "Scrobble from External Apps" card appears when at least one service is linked, with individual toggle switches
- New `GET /api/v1/subsonic/scrobble-settings` ŌĆö returns availability + current prefs
- New `POST /api/v1/admin/users/subsonic-scrobble` ŌĆö saves preferences (admin: any user; normal users: own account only)
- 6 new i18n keys (`player.subsonic.scrobble*`), all 12 locales
