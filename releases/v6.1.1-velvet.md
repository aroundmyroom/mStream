# v6.1.1-velvet — April 2026

Patch release fixing three bugs in the Album Library introduced in v6.1.0.

## Bug fixes

### fix: Albums view does not refresh after changing albumsOnly folder in Admin

When toggling the **Albums Only** flag on a folder via Admin → Directories, the server-side
source cache (`_sourcesCache`) was never cleared. The Album Library kept serving the old
folder's content until a full server restart.

Fix: `PATCH /api/v1/admin/directory/flags` now calls `invalidateCache()` from `albums-browse.js`
whenever the `albumsOnly` flag is changed, so the next page load immediately reflects the new config.

### fix: Tracks from a child vpath with spaces in its name returned 404

When the albumsOnly folder is a child vpath whose **name** contains spaces (e.g. `Unidisc 12-inch classics`),
track URLs were built as `/media/Unidisc%2012-inch%20classics/...`. Express's `express.static`
mount uses the literal vpath name with spaces, which does not match the percent-encoded URL,
causing every track to 404.

Fix: track filepaths are now built as `dbVpath + '/' + original_DB_filepath` (e.g.
`Music/12 inches/12 Inch Classics on CD (Unidisc Series)/Album/01.wav`). This routes through
the parent root vpath's static mount, which never has spaces in the URL segment.

### fix: albumsOnly source cache not cleared on config change

Related to the first fix — the sources cache listing which folders are albumsOnly sources
was indefinitely sticky across admin config changes within the same server session.
