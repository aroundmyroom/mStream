# v6.6.2-velvet — April 2026

Three improvements to the Admin Users panel: per-user file upload permission control, instant reactivity for all permission toggles, and consistent green/red colour coding across all permission pills.

---

## feat(admin): per-user file upload permission

A new **Upload** toggle is available per user in Admin → Users, alongside the existing Record and YouTube toggles.

| State | Colour | Meaning |
|---|---|---|
| ON (default) | Green | User can upload files via the file explorer |
| off | Red | Upload button hidden from the user; server rejects upload requests with 403 |

**Behaviour:**
- Applies to all users including admin accounts — there is no admin bypass
- The upload button in the file explorer disappears for the affected user immediately after their next page load (no server restart needed)
- Server enforces the flag at `/api/v1/file-explorer/upload` — a disabled user cannot upload even via direct API call
- `/api/v1/playlist/getall` exposes `allowUpload: bool` so the client hides the button without extra round-trips
- Global `noUpload` server setting still overrides everything (disables uploads for all users)

**Files changed:** `src/state/config.js`, `src/util/admin.js`, `src/api/admin.js`, `src/api/file-explorer.js`, `src/api/playlist.js`, `webapp/admin/index.js`, `webapp/app.js`

## fix(admin): permission toggles now update instantly without page refresh

All permission toggles in the Users and Directories panels now take effect immediately in the UI — no page refresh required.

**Root cause:** `getUsers()` and `getFolders()` used plain `object[key] = value` assignments, which bypass Vue 2's reactivity system. Subsequent `Vue.set()` calls inside toggle handlers had no reactive entry to update, so the template never re-rendered.

**Fix:** Both loaders now use `Vue.set(module.users/folders, key, value)` so every user and folder object enters Vue's reactive graph from the moment it is loaded. Additionally, `allow-upload`, `allow-radio-recording`, and `allow-youtube-download` are normalised to explicit booleans on load (missing key → correct default), preventing Vue 2 from missing `undefined → boolean` transitions.

---

## fix(admin): Record and YouTube permission pills now show green=ON / red=OFF

Previously the Record and YouTube toggles rendered as unstyled (greyed-out) when OFF and turned **red** when ON — the opposite of what the colour should mean.

All three permission pills (Record, YouTube, Upload) now follow a consistent rule:
- **Green** — permission is enabled
- **Red** — permission is disabled
