# v6.6.3-velvet — April 2026

Security and correctness fixes for the M3U playlist ZIP download endpoint.

---

## fix(security): M3U zip download path traversal

**Severity: High** — affected `POST /api/v1/download/m3u`

The endpoint that packages a `.m3u` playlist as a ZIP archive did not validate that each entry in the playlist resolved to a path inside the library root. A crafted playlist file containing relative path traversal sequences (e.g. `../../etc/shadow`) could cause the server to include arbitrary files from the filesystem in the returned archive.

**Fix:** each song path is now resolved with `path.resolve()` and checked against the library's `basePath` using the same trailing-separator bounds check already used elsewhere in the codebase. Entries that escape the root are silently skipped and a warning is written to the server log.

## fix: M3U fallback parser strips comment lines

The plain-text line-by-line fallback in the M3U reader (used when the `m3u8-parser` library returns no segments, e.g. for simple `.m3u` files) previously passed `#EXTM3U`, `#EXTINF:` and any other comment lines through as file paths. These would fail silently during archiving. They are now filtered out.
