### Radio Streams

A fully functional internet radio feature accessible from the player sidebar.

**Player UI (`webapp/app.js`, `webapp/style.css`)**
- New **Radio** section in the left navigation (Lucide radio icon, matches admin icon)
- Card grid layout (`auto-fill, minmax(155px,1fr)`) — same style as album art grid
- Each card shows station logo (with fallback icon), station name, and a play button
- Playing a station calls `Player.playSingle()` — correctly updates player bar title and artist
- A notice below the grid reads "Playing a radio stream clears the play queue" (`var(--t1)`)
- Drag-to-reorder stations (same drag model as playlists)
- Spectrum visualizer activates during radio playback

**Backend (`src/api/radio.js`)**
- `radio_stations` SQLite table: `id, user, title, url, img, sort_order, created_at`
- `GET/POST /api/v1/admin/radio/config` — admin enable/disable toggle
- `GET /api/v1/radio/enabled` — public check (authenticated users)
- `GET /api/v1/radio/stations` — list stations for current user
- `POST /api/v1/radio/stations` — add station (fetches + caches logo from URL)
- `PUT /api/v1/radio/stations/:id` — update station
- `DELETE /api/v1/radio/stations/:id` — delete + clean up cached logo
- `PUT /api/v1/radio/stations/reorder` — bulk sort_order update
- `GET /api/v1/radio/stations/:id/stream` — HTTP/1.1 stream proxy (required for ICY headers)
- `GET /api/v1/radio/stations/:id/nowplaying` — ICY `icy-title` now-playing extraction
- Art cached as `radio-{md5}.{ext}` in the album-art directory; SSRF-protected (no private IPs)

**Admin UI (`webapp/admin/index.js`, `webapp/admin/index.html`)**
- Radio Streams section under External Services
- Enable/disable toggle persists to `config.program.radio.enabled`
- Lucide radio icon in admin nav matches player nav
- Station list with inline add/edit/delete

**Docs**: `docs/API/radio.md` (full API reference), `docs/API.md` Radio section added

### Server-synced User Settings & Play Queue

All player preferences and the active play queue are now persisted in the database and restored on page load or across devices.

**`src/db/sqlite-backend.js`, `src/db/loki-backend.js`, `src/db/manager.js`**
- `user_settings` table: `(user TEXT PRIMARY KEY, prefs TEXT, queue TEXT, updated_at INTEGER)`
- Migration: created automatically on startup if absent

**`src/api/user-settings.js`** (new)
- `GET /api/v1/user/settings` — returns `{ prefs, queue }` for authenticated user
- `POST /api/v1/user/settings` — writes `prefs` and/or `queue`; partial updates supported

**`webapp/app.js`**
- `_collectPrefs()` — snapshot of all ~40 user preferences (volume, EQ, repeat, shuffle, themes, visualizer mode, etc.)
- `_syncPrefs()` — 1.5 s debounce; called at every pref-write site; flushes to DB
- `_syncQueueToDb()` — called only on structural queue changes (add/remove/reorder/song change)
- **`seeked` event**: debounced 1 s DB write after scrubber seek
- **15 s position-sync interval** during playback (enables accurate cross-device F5 restore)
- **`beforeunload`**: `navigator.sendBeacon` flushes exact position (own-browser F5 accuracy)
- DB is the source of truth on page load; localStorage is a fast local cache
- `restoreQueue`: seeked listener updates scrubber + waveform after position restore
- Debug timestamps in localStorage: `ms2_settings_pulled_<u>` / `ms2_settings_pushed_<u>`

**Docs**: `docs/API/user-settings.md` full API reference

### Decades & Genres Browser

A redesigned Browse section with dedicated Decades and Genres views, each with Albums and Tracks tabs, virtual scrolling, a sort bar, and a filter input.

**`webapp/app.js`**
- `viewDecades()` / `viewGenres()` — two-pane layout: drill-down list on left, content on right
- Albums tab: grouped album cards with artist + year; Tracks tab: virtual-scroll list of all matching tracks
- Sort bar: options vary by tab (album name/year/artist count for albums; title/artist/album/year for tracks)
- Browse filter: live text filter across the visible set
- Virtual scroll: renders only visible rows + overscan buffer for smooth performance on large libraries

**`src/db/sqlite-backend.js`, `src/db/manager.js`, `src/api/db.js`**
- `getDecades()` — returns decade buckets (1960s, 1970s, …) with track and album counts
- `getGenres()` — returns genre list with counts
- `getAlbumsByDecade(decade)` / `getAlbumsByGenre(genre)` — returns albums with cover art
- `getTracksByDecade(decade)` / `getTracksByGenre(genre)` — returns full track list, sorted

**`webapp/style.css`**: `.brow-*` CSS classes for the browse layout

**Docs**: `docs/browse-decades-genres.md` full feature reference

### Scan Error Repair Button

Scan errors now show a **Fix** button that attempts automatic file repair.

**`src/db/scanner.mjs`**
- FLAC: two-pass repair — `flac --decode | flac --encode`; also detects corrupt-frame errors
- MP3/WAV: lossless remux via ffmpeg (`-c copy`)
- Files confirmed unrecoverable (re-scan fails after repair attempt) are marked as such

**`src/api/admin.js`**
- `POST /api/v1/admin/db/scan-errors/fix/:hash` — triggers repair, re-scans file, returns result

**`webapp/admin/index.js`**
- **Fix** button per scan error row; spinner while repair runs; outcome reflected inline
- Gone-from-library indicator for files that no longer exist on disk
- `confirmed_at` chip showing when the error was last confirmed present

### Bug Fixes

- **Rate-song / scrobble fail for child-vpath files** (`45d9f071`): `filePath` lookup failed when the file lived under a child vpath; now resolves correctly
- **A-Z strip full-width layout** (`35a4d729`): A-Z letter buttons now use `flex: 1` and span the full content width instead of wrapping
- **Scan counter color**: `.spc-count` changed from `var(--t3)` to `var(--t1)` for legibility
- **Podcast Feeds view — complete UI overhaul** (`webapp/app.js`, `webapp/style.css`, `src/server.js`, `src/api/podcasts.js`, `src/db/*`):
  - Feed list was using the radio station tile grid (`.rs-list`) which rendered each feed as a compact square card, pushed the edit panel into a stray grid cell on top of the next card, and clipped art/text at wrong sizes.
  - Replaced with a dedicated vertical list layout (`.pf-list`, `.pf-item`, `.pf-feed-row`, etc.) — each feed is a full-width horizontal row: 88×88 cover art · title/author/stats/description · action buttons on the right.
  - Edit panel (`#pf-edit-panel-{id}`) is now inside its `.pf-item` wrapper and expands below its own row — no overlap with adjacent feeds.
  - Art images were permanently cached by the browser as a neutral SVG placeholder (the old `sendArtFallback` set `Cache-Control: public, max-age=86400`). Changed to `no-store` so the placeholder is never retained and the real image is served once downloaded.
  - Cache-busting `_v` parameter now derived from `last_fetched` (persists across page reloads) and stamped `Date.now()` after a feed refresh, ensuring the new art is always fetched.
  - Episode view header and player now-playing art also carry the `_v` buster via a new `album-art-v` song field.
  - `PATCH /api/v1/podcast/feeds/:id` extended to accept both `title` and `url`; `updatePodcastFeedUrl` added to SQLite backend, Loki backend, and manager. Edit panel now shows both display-name and RSS URL fields.
