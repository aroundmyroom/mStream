# v6.4.0-velvet — April 2026

Two quality-of-life improvements: the Loki in-memory database has been fully removed (the fork was originally cut from an experimental dual-backend branch), and admins can now set a server-wide default theme that new users see on first login.

---

## refactor(db): remove Loki backend — SQLite only

The fork was cut from an experimental branch that still carried the original LokiJS in-memory database as a fallback. Loki was never tested in production here and represented pure maintenance overhead — every DB feature had to be implemented twice, and the `db.engine` config option gave operators a false choice.

- Deleted `src/db/loki-backend.js` (1,428 lines)
- Removed `lokijs` from `package.json` (-3.1 MB from `node_modules`)
- `src/db/manager.js` — replaced runtime `if/else` engine branch with a static `import`; removed all `if (backend.x)` optional guards that existed only for Loki compatibility
- `src/state/config.js` — removed `engine` field from `dbOptions` schema; `.unknown(true)` added so existing configs with `"engine": "sqlite"` continue to boot without errors
- `src/api/admin.js` — removed `POST /api/v1/admin/db/engine` endpoint
- `src/util/admin.js` — removed `editDbEngine()` helper
- `webapp/admin/index.js` — removed "DB Engine" table row and modal
- `save/db/default.json` — cleaned up `db` block

**Migration note:** If your `default.json` has `"db": { "engine": "sqlite" }` it will continue to work — the unknown key is silently ignored. You can remove it manually if you wish.

---

## feat(admin): admin-configurable server-side default theme

Admins can now set the theme that users see before they have made a personal choice.

- New `ui` config key in `default.json`: `velvet` (default) | `velvet-dark` | `velvet-light`
- **Admin panel → Advanced → User Interface** — dropdown to select the default; takes effect immediately, no restart needed
- `/api/v1/ping` returns `defaultTheme` so the client applies the server preference right after login
- Users who have already chosen a theme personally are never affected — their stored preference always wins
