# v5.16.41-velvet — April 2026

## fix: yt-dlp 0-byte binary causes silent failure

### Problem
The Dockerfile `wget` download of yt-dlp could silently produce a 0-byte file (GitHub rate limit, redirect failure, CI network issue). The `chmod +x` step then ran on the empty file, leaving an empty-but-executable binary baked into the image. At runtime `_ensureYtdlp()` saw the file as present and executable (X_OK passed) and skipped re-downloading — so every yt-dlp call received an empty binary and failed silently.

### Fix

**Dockerfile**: validate download with `[ -s bin/yt-dlp/yt-dlp ]` (non-zero size check) before `chmod +x`. If the file is empty, `rm -f` it — this ensures the runtime auto-download path triggers cleanly on first use.

**`src/api/ytdl.js`**:
- After `X_OK` check passes, also verify `stat.size > 0`; if 0-byte, delete and fall through to download
- Reset `_ytdlpReady = null` on any download failure so the next request can retry
