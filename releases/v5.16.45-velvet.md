# v5.16.45-velvet — April 2026

## Scanner overhaul: 10–30x faster rescans + accurate progress bar

### Rescan speed
Rescans of unchanged libraries are now dramatically faster. Previously each file triggered an individual HTTP call + a separate SQLite transaction, making a 138K-file rescan take 2+ hours. Files are now processed in batches of 200 per HTTP call and all scanId updates are committed in a single SQL transaction per batch.

### Scan progress bar
- **Rescans**: start immediately — no pre-count walk, full I/O to scanning. Progress bar ticks within seconds.
- **First scans**: pre-count walk runs first with a live "Counting… N files found" display, then scanning begins with an accurate total.
- Progress percentage capped at 99% until the scan fully completes, preventing false "100%" when the DB count is stale.

### Last.fm gate
The Similar Artists toggle in Auto-DJ is now disabled when no Last.fm API key is configured.
