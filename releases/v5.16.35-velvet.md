# v5.16.35-velvet — April 2026

## Bug fix: Docker bootstrap admin login

### Problem
Setting `MSTREAM_ADMIN_USER` and `MSTREAM_ADMIN_PASS` environment variables in the
compose file did not result in a working login. The server reported
**"Login Failed. Check credentials"** even with correct credentials.

**Root cause:** The bootstrap used a single guard — *skip everything if folders are
already configured*. If the container was ever started before (which writes the
folder config), the user creation step was silently skipped on all subsequent
restarts, so the admin account was never written to `save/conf/default.json`.

### Fix
The bootstrap now runs two independent phases:

| Phase | Condition | Action |
|-------|-----------|--------|
| Folder | No folders in config yet | Creates `Music` folder (+ optional feature folders) |
| User | No users in config yet AND both `MSTREAM_ADMIN_USER` + `MSTREAM_ADMIN_PASS` are set | Creates the admin account with access to all configured folders |

Each phase runs at most once and independently. You can:
- Start with no admin vars (open mode), then add credentials on a later restart to lock it down
- Start with everything set at once — both phases run on first boot

### Feature folder default behaviour change
Previously, enabling `MSTREAM_ENABLE_RECORDINGS` (etc.) without a `*_SUBDIR`
variable would create an `Recordings` sub-folder inside `MSTREAM_MUSIC_DIR`.
Now it sets the folder type on `MSTREAM_MUSIC_DIR` itself — no unexpected
sub-folders are created. Use `MSTREAM_RECORDINGS_SUBDIR: MyFolder` to explicitly
put recordings in a sub-folder.

### How to get started fresh
If you have an existing `save/` volume with no users and want to add an admin:
1. Pull the new image: `docker compose pull`
2. Add `MSTREAM_ADMIN_USER` and `MSTREAM_ADMIN_PASS` to your `compose.yaml`
3. Restart: `docker compose up -d`
   — the user phase will run because no users exist yet, folders are unchanged.

If you want a fully clean start, remove the `save/` volume directory first.
