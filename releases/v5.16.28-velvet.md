### Bulk ZIP Download

Albums and playlists can now be downloaded as a single ZIP archive directly from the player UI.

- **ZIP button in the page header** — a small download icon + "ZIP" label appears when viewing an album or a playlist; automatically hidden on all other views (home, search, radio, podcasts, etc.)
- **Album downloads** — wired in `viewAlbumSongs()`; archive filename defaults to the album name
- **Playlist downloads** — wired in `openPlaylist()`; archive filename is the playlist name
- **Pre-flight size guard** — before streaming the ZIP the server checks the total uncompressed size of all requested files; if the total exceeds the configured limit, HTTP 413 is returned with `{ error, maxMb, sizeMb }` and the player shows a descriptive toast
- **Configurable max size** — new `scanOptions.maxZipMb` config field (default 500 MB); adjustable in Admin → DB Scan Settings → "Max ZIP Download Size"
- **Filename sanitisation** — the `filename` parameter is stripped of path-special characters and capped at 120 chars; defaults to `mstream-download` when absent
- **Frontend error handling** — 413 → toast with exact MB values; network errors → generic toast

**API:** `POST /api/v1/download/zip` — body: `{ fileArray: "[...]", filename: "My Album" }`; returns `application/zip` stream or `413 { error, maxMb, sizeMb }`

**Admin endpoint:** `POST /api/v1/admin/db/params/max-zip-mb` — body: `{ maxZipMb: 1000 }`

**Files:** `src/api/download.js`, `src/api/admin.js`, `src/util/admin.js`, `src/state/config.js`, `webapp/app.js`, `webapp/index.html`, `webapp/admin/index.js`

### Subsonic: getIndexes ifModifiedSince support

- **`ifModifiedSince` now honoured** — clients that send this parameter (DSub, Symfonium, etc.) receive an empty `indexes` response when the library has not changed since the given timestamp; eliminates a redundant full artist-list transfer on every poll
- **`lastModified` fixed** — previously always returned `Date.now()`, making every response look "fresh". Now reflects the actual last-scan timestamp (`MAX(ts)` from the files table)
- **`getLastScannedMs()`** — lightweight helper added to both SQLite and Loki backends (single `SELECT MAX(ts)` — no full stats scan)

**Files:** `src/api/subsonic.js`, `src/db/sqlite-backend.js`, `src/db/loki-backend.js`, `src/db/manager.js`

### Auto-DJ keyword filter — fuzzy double-letter matching

- **Bug fix** — filter words like `acapella` now correctly block songs with titles like `(Album Acappella)`. Both the filter word and the song title are normalised by collapsing repeated consecutive characters before matching, so spelling variants with single/double letters (`acapella` ↔ `acappella`, `accoustic` ↔ `acoustic`, etc.) are treated as equal. Case-insensitivity was already in place.

**Files:** `webapp/app.js`
