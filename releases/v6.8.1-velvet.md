# v6.8.1-velvet — April 2026 — Artist hydration startup bugfix

This patch fixes a practical issue in large libraries: Artist image hydration could appear stuck in **idle** even when many artists were missing images.

---

## Fix: explicit startup queue for artist hydration

After restart, hydration is only active when work is queued. In some sessions users saw:

- Hydration: idle
- Queue: 0 / 800
- Discogs: ready

but no visible progress.

### What changed

- Added new admin endpoint:
  - `POST /api/v1/admin/artists/hydration-seed`
- Added new Admin → Artists action:
  - **Queue next 500**
- Added UI guidance when queue is empty:
  - If hydration is idle and missing > 0, the panel now explains that no work is queued and offers the queue action.

### Result

- Admins can start processing immediately without waiting for browse routes to enqueue artists.
- Status transitions are visible right away (queue/running state updates after seeding).
- Better first-run and post-restart behavior for libraries with very high missing-image counts.
