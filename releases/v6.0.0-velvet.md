# v6.0.0-velvet — April 2026

This is a milestone release. The version number reflects the stability and
feature maturity the fork has reached since branching from mStream 5.14.3.

---

## Fix: scanner no longer stalls after the first 200 files

**Root cause:** `findFilesByPaths()` called `db.exec('BEGIN')` while the
`add-file` endpoint already had an open transaction. SQLite raised "cannot
start a transaction within a transaction" → HTTP 500 → "Batch lookup failed"
→ scanner stopped dead after exactly one batch.

- `findFilesByPaths()` — removed unnecessary `BEGIN/COMMIT` (reads don't
  need a transaction)
- `batchUpdateScanIds()` — switched from `BEGIN/COMMIT` to `SAVEPOINT` /
  `RELEASE` / `ROLLBACK TO` (nestable inside any open transaction)

## Feat: "added to DB" counter in the scan progress card

The scan progress card in the admin UI now shows two lines during a scan:

- **`N files checked`** — files looked up via the batch endpoint
- **`N added to DB`** *(green)* — files actually inserted into the database,
  updating in real time

## Fix: `/login` redirects to Velvet UI — old login page removed

- `GET /login` and `GET /login/` now return a permanent 301 redirect to `/`
- `webapp/login/` directory removed entirely — it was dead code serving the
  old upstream mStream logo and login form
- The `/admin` unauthenticated redirect was also updated from `/login` to `/`
