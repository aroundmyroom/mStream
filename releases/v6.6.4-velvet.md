## fix: Subsonic stream Content-Type â€” FLAC and WAV now play on iOS

Express's built-in MIME module maps `.flac` â†’ `audio/x-flac` and `.wav` â†’ `audio/x-wav`.
iOS AVFoundation only recognises the IANA-registered types `audio/flac` and `audio/wav`
and silently refused to play those files, causing the client to retry the stream every
~12 seconds in a loop.

The `stream` endpoint now sets `Content-Type` explicitly from the DB `format` field
using the same MIME map used in `buildSong`, ensuring iOS (iSub, Sonora) can play every
format supported by the server.

## fix: Subsonic getCoverArt honours `?size=` â€” fixes first-play art race on iOS

`getCoverArt` previously ignored the `?size=` parameter and always served the full-
resolution image regardless of what the client requested. For large images (some > 3 MB)
the image had not finished transferring by the time the stream started (~8 ms later), so
iOS never showed art for the first play; subsequent plays used the client cache.

**Fix:** when `size > 0`, the handler now serves a pre-generated thumbnail:
- `size â‰¤ 92` â†’ `zs-<hash>.<ext>` (92 px, ~20 KB)
- `size > 92` â†’ `zl-<hash>.<ext>` (256 px, ~100â€“150 KB)

Thumbnails are already generated by the scanner. A one-time backfill script was run to
generate the `zl-`/`zs-` versions for the 316 existing art files that were missing them
(5 corrupt files could not be processed).

`getCoverArt` also now adds `Cache-Control: public, max-age=86400` so clients cache
art across sessions.

## fix: Subsonic getAlbumList Recently Added/Played shows correct song count and duration

`getAlbumList?type=newest` and `type=recent` return song rows, not pre-aggregated album
rows. The album objects built from those rows always had `songCount: 0` and no duration,
which Sonora displayed as "0.01 sec".

**Fix:** after deduplicating by `album_id`, a single `getAlbumStatsByIds()` query
fetches `COUNT(*)` and `SUM(duration)` for all albums at once and injects them into
the album objects.  `getAlbumsByArtistId` and `getAllAlbumIds` also now include
`SUM(duration)` so the artist view benefits too.

## feat: Subsonic getNowPlaying implemented

`getNowPlaying` previously returned an empty list. It now tracks which song each user is
currently playing via the `scrobble?submission=false` notification that clients send when
playback starts:

- Entries expire after 30 minutes
- Returns full song metadata + `username`, `minutesAgo`, `playerId`, `playerName`
- Completed plays (`submission=true`) clear the now-playing entry

## fix: Subsonic stream accepts Sonora preview IDs (`<hash>-preview-0`)

Sonora (OpenSubsonic client) appends `-preview-N` to song IDs when requesting a preview
stream. The suffix caused the DB hash lookup to fail with a 404. The stream handler now
strips `-preview-N` before the lookup, serving the full file (no clip is generated).

## fix: Subsonic getPlaylist resolves child-vpath song entries

Playlist entries created by the web client while browsing a child vpath (e.g.
`12-inches`) were stored with the child vpath name in the filepath prefix. The Subsonic
`getPlaylist` handler looked them up as-is, which always failed because the DB only
stores files under the root parent vpath (`Music`).

**Fix:** the handler now consults `vpathMeta` per entry â€” if the stored vpath is a
child, it remaps to `parentVpath + filepathPrefix + relative_path` before the DB lookup.
