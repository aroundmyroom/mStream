# v6.3.0-velvet — April 2026

## What's changed

### Feat: Server Audio — mpv plays through the server's speakers

mStream can now play music **directly through the server's audio hardware**. Any browser that can reach your server becomes a remote control — no app install needed.

**How it works:**  
mStream spawns [mpv](https://mpv.io/) in idle mode and controls it over a local Unix socket (JSON IPC). The browser remote polls playback state every 2 seconds and sends commands through the existing authenticated REST API.

**Remote control page — `/server-remote`**  
A self-contained single-page app with:
- Now Playing bar (art, title, artist, progress seek bar)
- Transport controls: play/pause, prev, next, loop cycle (off → one → all)
- Volume slider (0–130, mpv software boost supported)
- **Queue tab** — list all queued songs, tap to jump, × to remove
- **Auto-DJ tab** — keeps music playing automatically; two modes:
  - *Random* — picks random songs from the library, avoids recently played tracks
  - *Similar Artists* — uses the Last.fm similar-artists API for context-aware selection (requires Last.fm API key in Admin → Last.fm)
- **Browse tab** — file-explorer for your vpaths, "+ Queue" button per song
- Login screen for installs with users; auto-authenticates on open installs
- Three themes: Velvet / Dark / Light

**Admin Panel — Server Audio section**  
New section in Admin → Config → Server Audio:
- **Enable toggle** — auto-starts mpv when mStream boots
- **mpv Binary Path** — configure a custom path (default: `mpv` on PATH)
- **Detect** button — verifies the binary and shows its version
- **Start / Stop** — manually control the mpv process
- **Open Remote** link — opens `/server-remote` in a new tab

**New config keys** in `save/db/default.json` under `serverAudio`:
```json
{
  "serverAudio": {
    "enabled": false,
    "mpvBin": "mpv"
  }
}
```

**REST API** (all auth-required except the remote page itself):
- `GET /api/v1/server-playback/status`
- `GET /api/v1/server-playback/detect`
- `POST /api/v1/server-playback/pause|next|previous|seek|volume|loop`
- `POST /api/v1/server-playback/queue/add|clear|remove|play-index`
- `GET/POST /api/v1/admin/server-audio`, `/start`, `/stop`

See [`docs/server-audio.md`](../docs/server-audio.md) for full setup, install instructions, API reference, and troubleshooting.

### Fix: Jukebox remote addSong 404 on child vpaths with spaces

When adding a song via the Jukebox browser remote file browser, the filepath was built from the raw child-vpath name (e.g. `Unidisc 12-inch classics/...`). Express.static mounts use literal vpath names; browsers percent-encode spaces → Express couldn't match the mount → 404.

Fixed in `webapp/app.js`: the `addSong` WebSocket handler now resolves the correct DB parent-vpath filepath via `api('POST', 'api/v1/db/metadata', { filepath })` → `norm(meta)` before queuing. Falls back to the raw path for non-indexed files.
