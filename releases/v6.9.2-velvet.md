# v6.9.2-velvet — April 2026 — Translation fixes & recording deletion fix

`v6.9.2` is a patch release with additional missing translation fixes and a bug fix for incorrect song playback after deleting a recording.

---

## Bug fix: wrong song played after deleting a recording

When a recording was deleted from the **Recently Added** (or any song list view), the remaining rows kept their original `data-ci` index attributes. Clicking a song _after_ the deleted position would look up the wrong index in `S.curSongs` and play an incorrect song.

**What was wrong:**
- Row removal used a fragile title-text match (`querySelector` by `.song-title` textContent) — could remove the wrong row if two songs had the same title.
- After removal, no `data-ci` renumbering was done — all remaining rows and their inner elements (add-button, context-button, stars) kept stale indices.

**Fix:**
- Row is now removed via exact `data-ci` selector: `.song-row[data-ci="${idx}"]` — always correct.
- After splice and DOM removal, all `[data-ci]` elements with a value greater than the deleted index are decremented by 1, keeping the DOM and `S.curSongs` array in sync.

## Additional missing translations fixed

Continued translation sweep — all strings were still rendering in English regardless of selected language.

**Auto-DJ info strip:**
- "Similar to **{artist}**" → `player.autodj.stripSimilarTo`
- "We will play:" → `player.autodj.stripWillPlay`
- "Other choices were:" → `player.autodj.stripOtherChoices`
- "Queue restored · N song(s)" → `player.autodj.stripQueueRestored` + existing plural key

**Album detail view:**
- "▶ Play" button → `player.ctrl.albPlayAll`
- "+ Add to Queue" button → `player.ctrl.albAddAll`
- "Clicking a track plays that disc from that track." hint → `player.ctrl.albPlayHint`
- Multi-disc variant with disc-tab tip → `player.ctrl.albPlayHintDisc`
- "Choose a Recordings folder to save to…" paragraph in recording modal → `player.modal.recChooseFolder`

**Radio recording button:**
- "Record this stream" → `player.ctrl.recordStream` (key existed, was never wired)
- "Stop recording" → `player.ctrl.stopRecording` (new key)

**Custom tooltip system fix:**
- `MutationObserver` now watches `title` attribute mutations in addition to added nodes.
- All player control buttons using `data-i18n-attr="title"` (play, pause, next, prev, shuffle, repeat, expand NP, EQ, sleep timer, etc.) now correctly show the styled `#tip-box` tooltip instead of the browser's native tooltip.
