# v6.13.1-velvet — April 2026 — Artist Albums Diagnostic

## Overview

v6.13.1 is a focused Admin tooling patch. It adds a self-service **Artist Albums Diagnostic** panel in the Admin UI — designed to help Docker users (who have no shell access) investigate why certain albums are missing from an artist's page without needing to run any scripts.

---

## New: Artist Albums Diagnostic tool (Admin UI)

### The problem

Albums can disappear from the Artist view while remaining findable via Search. This happens when the `COALESCE(album_artist, artist)` value stored in the database for a track does not exactly match any of the artist's **raw variants** in the normalised artist index. The mismatch can be as subtle as a letter-case difference, a trailing punctuation mark, or a "Mixed by X" prefix that the normaliser did not absorb.

Previously, diagnosing this required direct SQLite access — not possible from within a Docker container.

### What the tool does

Navigate to **Admin → Artist Albums Diagnostic**, type an artist name, and click **Run Diagnostic**. The panel returns:

| Section | What you see |
|---|---|
| **Summary banner** | Total distinct effective-artist values · Covered by normalised variants · Orphaned values · Hidden albums |
| **Normalized Index Entry** | Canonical name, song count, known vpaths, full list of rawVariants that the artist view queries |
| **Orphaned artist values** (red card) | Every effective-artist value that exists in the DB but is NOT in rawVariants — with the albums and directories they hide |
| **Albums per variant** | All visible albums, broken down by which rawVariant matched them |

### Export options

Three export buttons appear once results are loaded:

- **↓ Export .txt** — downloads a human-readable plain text report (easy to paste into a support thread or forum post)
- **↓ Export .json** — downloads the full raw JSON response
- **Copy JSON** — copies to clipboard with a 2-second "✓ Copied!" confirmation

Files are named automatically from the artist query, e.g. `artist-diag-Ben_Liebrand.txt`.

### How to fix a missing album

The "Orphaned artist values" card shows the exact tag value stored in the DB and the affected albums. To restore visibility:

1. Retag the `album_artist` field on those files so it matches one of the **rawVariants** shown in the Normalized Index Entry (or matches the canonical name exactly).
2. Re-scan the library.
3. Run the diagnostic again — the orphaned card should be empty.

### API endpoint

```
GET /api/v1/admin/diagnostics/artist-albums?artist=<name>
```

Admin-only (requires `x-access-token` with admin user). Returns:

```json
{
  "query": "Ben Liebrand",
  "normalizedEntry": { "canonicalName": "...", "rawVariants": [...], "vpaths": [...], "songCount": 0 },
  "effectiveArtistValues": [...],
  "albumsByVariant": { "Ben Liebrand": [...] },
  "orphanAlbums": [{ "effective": "mixed by Ben Liebrand", "track_count": 31, "albums": [...] }],
  "summary": { "totalEffectiveValues": 9, "coveredByVariants": 1, "orphanedValues": 8, "orphanedAlbumCount": 31 }
}
```
