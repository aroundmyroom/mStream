# v6.12.0-velvet — Going Back to My Roots

**Release date:** April 19, 2026

> *"Going Back to My Roots"* — In the early 2000s, every NAS shipped with Twonky or Serviio, and any TV
> or AV receiver on the home network could instantly browse and play your music collection without
> installing anything. mStream Velvet brings that experience back.

---

## New feature: DLNA / UPnP Media Server

mStream Velvet now includes a built-in **UPnP ContentDirectory MediaServer (DMS-1.50)**.

Enable it in **Admin → DLNA / UPnP** and within seconds your music collection appears on every
smart TV, AV receiver, NAS player, VLC instance, and BubbleUPnP client on your LAN — no account,
no HTTPS, no app install required.

### How it works

- A dedicated **plain-HTTP server** (default port **10293**) runs alongside the main mStream HTTPS
  server. DLNA devices require plain HTTP — they cannot handle HTTPS or JWT tokens.
- **SSDP multicast** advertises the server automatically so devices discover it without typing an
  IP address.
- Devices send `Browse` SOAP requests and receive a **DIDL-Lite folder tree** mirroring your real
  filesystem.
- Audio files are streamed directly in their **original format** — no transcoding. FLAC, MP3, Opus,
  WAV, M4A all served with correct Content-Type headers.
- **Album art** served from the mStream art cache.

### Browse hierarchy — real folder tree

The browse hierarchy mirrors the actual filesystem of your `albumsOnly` vpaths — exactly as a
classic NAS DLNA server would show it:

```
Root
├── Albums
│   ├── -= Top 700 =-
│   │   ├── 01. Ring My Bell.flac
│   │   └── …
│   ├── Disconet
│   │   ├── Vol 1
│   │   │   └── …
│   │   └── Vol 2
│   │       └── …
│   └── …
└── Disco
    └── …
```

- **Albums shown = your `albumsOnly` vpaths** — the same sources as the Album Library. Only proper
  albums appear; the 100,000+ random files in non-album folders are never visible to DLNA clients.
- **Unlimited folder depth** — CD box sets, vol/series sub-folders all browseable one level at a time.
- **Songs sorted by track number**, then filepath.
- **5-minute in-memory cache** per source — fast browsing without hitting the DB on every request.

### Configuration

| Setting | Default | Description |
|---------|---------|-------------|
| Enabled | `false` | Toggle live — no server restart |
| HTTP Port | `10293` | Dedicated plain-HTTP port for DLNA |
| Server Name | `mStream Velvet` | Friendly name shown on TV source lists |

All changes take effect immediately (no restart needed).

### Supported SOAP actions

`Browse` (BrowseDirectChildren + BrowseMetadata with full pagination) ·
`GetSystemUpdateID` · `GetSearchCapabilities` · `GetSortCapabilities` ·
SUBSCRIBE/UNSUBSCRIBE stub accepted

### Security note

The DLNA port is **intentionally unauthenticated** — DLNA devices have no concept of user accounts.
Only enable on a trusted private LAN. Never port-forward port 10293 to the internet.

### Dependency

`node-ssdp` v4.0.1 (MIT licence) — SSDP advertisement layer only. No new runtime dependencies for
the media serving or SOAP handling.
