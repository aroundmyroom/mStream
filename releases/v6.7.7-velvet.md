# v6.7.7-velvet — April 2026 — Waveform pre-fetch + crossfade dissolve

## Feat: waveform pre-fetch — no "Generating waveform…" delay in Auto-DJ

Waveforms were previously generated the moment a song started playing. In Auto-DJ
mode the next song has typically been queued for 30+ seconds before it plays, so
the delay was entirely unnecessary — the waveform could have been ready in advance.

A silent background pre-fetch queue (`_wfPrefetchPending` / `_wfPrefetchDrain`)
now processes one request at a time (300 ms apart) whenever songs are added to
`S.queue`. It is hooked into every queue-add path:

- `Player.addSong` / `Player.addAll` / `Player.playNext` / `Player.setQueue`
- Both Auto-DJ code paths (`autoDJPrefetch`, `autoDJFetch`)

Each prefetch hits the real `GET /api/v1/db/waveform` endpoint. The server writes
`wf-{hash}.json` to `waveformDirectory` and the client caches the result in
localStorage — both caches are warm before the song begins playing, giving instant
display with no generation spinner.

## Feat: waveform crossfade dissolve — synced with album art and title

The waveform bar chart was previously swapped abruptly mid-crossfade, out of sync
with the smooth art and title transitions.

The waveform now dissolves in perfect sync with the rest of the player UI:

- A canvas snapshot of the **outgoing waveform** is captured at crossfade start
  and laid over the waveform area as `.xf-wf-out`; it fades out over the full
  crossfade duration
- The **incoming waveform** (if already pre-fetched) is painted onto `.xf-wf-in`
  at `pct=0` and fades in simultaneously — same duration, same moment as the
  album art and title/artist text
- Both overlays share a single `offsetHeight` flush so their CSS transitions start
  in perfect sync
- If the incoming waveform is not yet cached, it appears via `_wfFadeIn()` (0.45 s
  ease-in) once generation completes — no abrupt pop

Additional waveform polish:
- All new waveforms always fade in (0.45 s `ease-in`) rather than appearing
  instantly; canvas resets to `opacity: 0` when clearing so stale data never
  bleeds through
- Extracted `_drawWaveformBars(ctx, W, H, data, pct)` as a pure drawing helper
  shared by the live canvas and both overlay canvases
