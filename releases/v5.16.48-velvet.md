# v5.16.48-velvet — April 2026

## fix: scanner no longer hangs indefinitely on corrupt/unreadable files

During a first-time scan, the scanner could stall after exactly one batch (200 files) and never continue.

**Root cause:** `parseFile()` from music-metadata and `calculateHash()` can hang forever on a corrupt, malformed, NFS-stalled, or otherwise unreadable file. The scanner child process was stuck waiting with no timeout, blocking all subsequent files.

**Fix:** Added a `withTimeout(promise, 30000)` helper that races any async operation against a 30-second deadline. Both `parseFile` and `calculateHash` are now wrapped. A file that exceeds the timeout is logged as a `parse` scan error and skipped; scanning resumes immediately with the next file.
