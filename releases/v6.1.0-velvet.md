# v6.1.0-velvet — Complete Album Library

> Released: April 2026

## Overview

This release delivers a complete, ground-up rebuild of the album browsing experience. The former flat "Albums" list is replaced by a fully DB-driven Album Library with automatic series grouping, multi-disc support, intelligent cover art resolution, and a fast in-memory cache.

---

## What's new

### Album Library (complete new system)

**DB-driven, no filesystem walking.** The entire album tree is built in memory from the already-indexed `files` table — first load takes ~60 ms regardless of library size. Subsequent loads are instant (5-minute cache).

**Automatic structure detection** from folder layout:

- **Standalone album** — files directly inside `Albums/<folder>/`
- **Multi-disc album** — sub-folders matching disc naming patterns (`CD 1`, `Disc 2`, bare numeric `1`, `2`) become disc tabs inside one album
- **Series** — sub-folders that are *not* disc folders become individual album cards grouped under a series banner (e.g. a "Megamixes" folder containing Vol. 1, Vol. 2, Vol. 3)

**Cover art resolution** (in priority order):
1. On-disk image file in the album folder (`cover.jpg`, `cover.png`, `front.jpg`, `Folder.jpg`, `cover.webp`, and case variants)
2. On-disk image inside the first disc sub-folder (for multi-disc albums)
3. Embedded art from the database (`aaFile`), extracted at scan time

**Frontend views:**
- **Library grid** — series cards and standalone album cards with live search filter
- **Series drill-down** — all sub-albums in a series
- **Album detail** — disc tabs, full track list, per-track and batch play/queue controls
- **Play behaviour hint** — a small note below the disc tabs explains that clicking a track loads the full album (all discs) into the queue starting from that track

### Queue — disc separators

When a multi-disc album is loaded into the queue, the queue panel now shows a labelled divider between discs (e.g. **CD 1**, **CD 2**) so it's always clear which disc each track belongs to. Single-disc albums and non-album queues are unaffected.

**New API endpoints:**
- `GET /api/v1/albums/browse` — returns full `{ albums, series }` tree
- `GET /api/v1/albums/art-file?p=` — serves on-disk art files (path-traversal safe)

### Navigation cleanup

- Old "Albums" nav button (flat list / `db/albums` backed) removed
- New Album Library button takes its place, labelled "Albums"
- `viewAllAlbums()` function removed from client code; back-navigation from artist album grids now returns to the Album Library
