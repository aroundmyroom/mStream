# v6.2.0-velvet — April 2026

## Your Stats — on-server listening statistics

A privacy-first, fully local listening history and statistics system. All data stays in your own SQLite database — no external calls, no telemetry.

### What's tracked
- **Songs**: every play, skip, and natural end is recorded with timestamps and duration
- **Radio**: station listens — start, stop, ICY now-playing title at session start
- **Podcasts**: episode plays — feed URL, episode ID, title, duration

### What you see
Open **Your Stats** from the sidebar navigation. Use the period picker to browse:

- **Week / Month / Quarter / Half-Year / Year** with ← / → navigation
- **Summary strip**: total plays, unique songs, total listening time
- **Top Songs / Top Artists / Top Albums** with album art
- **Listening clock**: hour-of-day and day-of-week bar charts
- **Personality card**: inferred listening archetype (Night Owl, Explorer, Loyalist, …)
- **Radio card**: top stations and total radio time (when radio has been used)
- **Podcast card**: top shows and episode count (when podcasts have been used)
- **Fun facts**: longest session, most-played day, favourite genre, and more

### Admin tools
**Admin → Play Stats** shows:
- Total event count and estimated DB storage
- Per-user breakdown table
- Per-user purge tool (keep last N months of data)

## Bug fixes

### Radio "in library" detection — fuzzy multi-strategy matching
ICY StreamTitle metadata (e.g. `"ELIZA ROSE & INTERPLANETARY CRIMINAL - BOTA (BADDEST OF THEM ALL)"`) is now matched against the library using a smarter strategy:

- Split on ` - ` and search each half independently (handles both `ARTIST - TITLE` and `TITLE - ARTIST` orderings)
- Strip special characters (`&`, `(`, `)`, `.`) before querying FTS5 — prevents parse errors
- Extract parenthetical content as an additional search candidate, covering abbreviation mismatches like `BOTA` vs `B.O.T.A.`

### Search — clicking an album showed "No songs found"
Clicking an album in search results incorrectly applied the albumsOnly path filter, hiding songs stored outside the designated albums subfolder. Fixed — search results now open albums with the full unfiltered scope.

### Artist page — incomplete album list
The artist albums page was applying the albumsOnly filter, hiding releases in non-albumsOnly folders (Minimixes, live recordings, individual singles, etc.). Now consistent with the artist list view — all releases shown.

### Your Stats — stats period in wrong timezone
Period boundaries (week start, month start, etc.) were calculated in UTC instead of the server's local time. Fixed in both the stats aggregation and the period list query.

### Your Stats — period offset navigation broken
The `offset` parameter was treated as a string, causing `month + "1"` → `"month1"` string concatenation for offsets > 0. Fixed with explicit `parseInt()`.

### version tooltip on sidebar logo
`/api/v1/ping` now returns the running version; hovering the sidebar logo shows it as a browser tooltip.
