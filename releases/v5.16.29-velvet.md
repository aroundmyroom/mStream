# v5.16.29-velvet

## What's new

### Allow users to delete their own recordings (admin-gated)

A new per-folder permission lets admins choose whether users may delete recordings they own.

**Admin panel**
- When a folder is configured as a *Radio Recordings* folder a new sub-option appears beneath it: **"Allow users to delete their own recordings"**.
- The sub-option is hidden (and reset) whenever the recordings checkbox is cleared.

**Context menu**
- A **Delete Recording** item (red, with divider) appears in the song context menu *only* for songs coming from a recordings folder that has the permission enabled.
- Clicking it asks for confirmation ("This cannot be undone.") before proceeding.
- On success: the file is deleted from the server, the row is removed from the current view, the track is removed from the queue (if present), and a toast confirms the action.

**API**
- New `DELETE /api/v1/files/recording` endpoint ŌĆö requires authentication, validates the filepath, confirms the vpath is of type *recordings* with `allowRecordDelete` enabled, checks the extension against the supported audio file list, then removes the file from disk.

### Performance improvements

- `qr.js` is now loaded with `defer` ŌĆö no longer blocks first paint.
- `checkSession()` fetches independent status endpoints in parallel (`Promise.all`) instead of serially.
- `visibilitychange` handler has a 30-second cooldown to avoid redundant API bursts on repeated tab switches.
- Virtual-scroll handlers (inline albums, album grid, song list) are guarded by `requestAnimationFrame` + a pending-flag to cap DOM writes at one per frame.
- Queue panel event listeners moved from per-element (N ├Ś 5 on every render) to 5 delegated listeners on the container.
- `contain: content` on `.content-body` and `contain: strict` on `.vslist-wrap` reduce browser layout recalculation during scroll.

### Anonymous Telemetry

mStream Velvet now sends a small anonymous ping once at boot (60 s delay) and every 24 hours to a private Cloudflare Worker. This helps track active installations and version distribution ŌĆö no personal data is collected.

**Payload sent:**
```json
{ "id": "<random UUID>", "version": "5.16.29-velvet", "platform": "linux", "lastSeen": 1774595289943 }
```

- UUID generated once, stored in `save/conf/instance-id` ŌĆö not linked to any user, hardware, or IP address
- No IP addresses, usernames, file paths, or any personal data collected or stored
- **Opt out:** add `"telemetry": false` to your config and restart

See [docs/telemetry.md](../docs/telemetry.md) for full details.

### Admin Backup Panel

A new **Backup** section in the admin nav (archive icon) lets admins create and download zip archives of the database and config.

- Zips the active database + `default.json` into `save/backups/`
  - SQLite: `mstream.sqlite` + WAL/SHM files if present
  - Loki: all three `.db` files
- Keeps the 4 most recent backups ŌĆö oldest removed on fifth
- Manual "Create Backup Now" button; list shows filename, size, date, download
- **Automatic weekly backup** ŌĆö checked hourly with a 30 s boot delay

### Radio Now-Playing DB Badge

When a radio stream shows an artist/title, a small database icon appears next to the text if the song is found in the local library.

- 3-second debounce after track change before the search fires
- Badge and timer cleared on track change or when leaving radio
