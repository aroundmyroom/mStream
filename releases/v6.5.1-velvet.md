# v6.5.1-velvet — Bug-fix release

Released: April 2026

## What's in this release

This is a bug-fix and polish release on top of v6.5.0-velvet. All features from v6.5.0 are included.

---

## Bug fixes

### fix(admin): purge endpoint returned 403 when `keepMonths: 0` was sent

The original play-stats purge schema used `keepMonths` with `min(1)`, causing a Joi validation error (mapped to HTTP 403) when users tried to delete all data. The endpoint has been redesigned:

- New `fromMs` / `toMs` parameters replace `keepMonths` — delete a precise time window instead of "keep last N months"
- Minimum value is `0` (epoch), so any valid timestamp is accepted
- Radio and podcast play events are now also purged in the same range operation

### fix(admin): backfill endpoint returned 500 (`db.transaction is not a function`)

`node:sqlite`'s `DatabaseSync` does not have a `.transaction()` helper (that is a `better-sqlite3` API). The backfill function now uses `db.exec('BEGIN')` / `db.exec('COMMIT')` which is the correct pattern for this codebase.

### fix(admin): HTML tags rendered as literal text in confirm dialog

Vue's `{{message}}` interpolation HTML-escapes its value. The confirm modal's `<p>` now uses `v-html="message"` so bold text and line breaks render correctly.

## New features (also in 6.5.0)

### feat: Folder-name metadata fallback for untagged files

Files with no embedded tags (artist/album/title all null) now get values derived from the parent folder name, using the pattern **"Artist - Release info"**:

- **Scanner** — applied automatically for new or modified files during every scan
- **Admin backfill** — "Fix Missing Metadata (DB only)" card in the Play Statistics admin tab runs a one-shot update for all existing null-artist rows and rebuilds the artist search index

**Important:** this is a **database-only** operation. Audio files on disk are never modified. Derived values survive rescans: unchanged files are not re-parsed and their DB values are preserved.

See [docs/metadata-fallback.md](../docs/metadata-fallback.md) for full details.
