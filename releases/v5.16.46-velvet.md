# v5.16.46-velvet — April 2026

## fix: scanner batch lookup 500 errors

Rescans on libraries with many unchanged files were producing "Batch lookup failed: Request failed with status code 500" errors in the scan error audit.

**Root cause:** `findFilesByPaths()` constructed a dynamic `IN (?,?,?…200 placeholders)` SQL string and called `db.prepare()` inside the HTTP request handler. Node.js's built-in `node:sqlite` `DatabaseSync` module throws when preparing statements this way at runtime, causing the batch endpoint to crash with a bare 500.

**Fix:** The cached `_s.findFile` prepared statement is now reused inside a read transaction loop — same batched-read performance, no dynamic SQL, no variable-number limit. The `get-files-batch` endpoint is also wrapped in try/catch so any future DB error returns a proper JSON error response.
