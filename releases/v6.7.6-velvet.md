## Fix: dice faces show through each other after browser update

`backface-visibility: hidden` was missing from `.dj-dice-face`, so browsers that
enforce the CSS 3D spec correctly were rendering back-facing die faces as
transparent rather than invisible â€” each visible face was composited on top of
whatever was behind it.

- Added `backface-visibility: hidden` to `.dj-dice-face`
- Reduced `border-radius` from 10 px to 4 px to close the corner gap exposed
  once back-face hiding is active
- `_throwDjDice`: cancel all running animations via `getAnimations()` instead of
  only the two stored refs (`_djw` / `_djc`) â€” orphaned fade-out animations
  accumulated across multiple throws and composited against new ones, causing
  ghost frames

## Fix: album art search smarter for catalog numbers, "Presents" artists, and compilation titles

### Catalog number search (Discogs)

`extractCatno()` now parses catalog numbers from folder names and album tags â€”
formats like `(SMR 624)` or `[SP5-1306]`. Discogs `catno=` is a dedicated index
field with essentially no false positives.

Two new Phase A searches run in parallel with the existing ones:
- **A6a** â€” `catno + release_title`
- **A6b** â€” `catno` alone (for title mismatches)

Example: `Various - Hit Mix '86 (2xLP 1986) (SMR 624)` now finds the exact release
immediately instead of falling back through multiple fuzzy searches.

### "Presents" artist handling

New `presentsLabel()` helper detects artist strings like `"Salsoul Presents"` and
extracts the label name (`"Salsoul"`).

- Three new Phase A Discogs searches: **A7a** (album title only), **A7b**
  (label + album title), **A7c** (master by album title)
- `artistMatchScore` awards 0.85 to Discogs "Various" results when the input
  artist contains "Presents"
- iTunes and Deezer server proxies strip the "Presents" prefix before querying
- Example: `Salsoul Presents - The Definitive 12'' Masters Vol. 2` now resolves
  to `Various â€“ The Definitive 12" Masters Vol. 2`

### Deezer proxy update

`GET /api/v1/deezer/search` now accepts `?artist=&album=` separated params in
addition to `?q=`, allowing the server to apply Presents-stripping and build more
targeted queries.

## Fix: Discogs misses albums where the artist name has extra words on Discogs

Artists tagged as e.g. "Alfie Khan" but credited on Discogs as "Alfie Khan Sound
Orchestra" failed all Phase A searches because every existing search uses
`type=release` with a strict artist-field filter.

- Added **A1m**: `type=master, artist, release_title, year` â€” master searches are
  fuzzier about artist name credits
- Added **A4m**: `type=master, q="artist album [year]"` â€” free-text master search;
  master result sets are smaller so the correct album ranks higher
- Both run in Phase A (parallel with existing searches) â€” no slowdown for
  normal cases

## Fix: plays on unindexed files are now tracked

Files played via the file explorer before a full library scan had been run used to
silently drop `play-start` and `log-play` events â€” the DB lookup returned null and
the event was discarded.

New `src/util/on-demand-index.js`: when a played file has no DB row, it is indexed
immediately â€” an MD5 hash (first 512 KB), metadata parsed via `music-metadata`, and
a minimal `files` row inserted â€” before play-tracking proceeds.

- Applies to `POST /api/v1/wrapped/play-start` (Wrapped stats) and
  `POST /api/v1/db/stats/log-play` (recently-played / play-count)
- Both endpoints converted to `async` handlers
- On-demand rows are transparent â€” they appear in the library and merge cleanly
  with the next full scan
