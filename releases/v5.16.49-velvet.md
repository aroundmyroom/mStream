# v5.16.49-velvet — April 2026

## fix: Scan Error Audit page no longer crashes with thousands of errors

`GET /api/v1/admin/db/scan-errors` returned ALL rows with no limit. With 82,000 accumulated errors the JSON response was massive and the page failed to load entirely, making it impossible to see what was actually wrong.

**Fixes:**
- Endpoint now returns `{ errors: [...first 500 rows], total: N }`. A `?limit=N` query param allows up to 5000 rows.
- The Fix button endpoint (`POST .../fix`) was loading all errors just to find one by GUID — replaced with a direct single-row lookup.
- UI shows a yellow warning banner when results are truncated: *"Showing first 500 of N errors — use Clear All to remove old entries, then re-scan to re-detect current problems."*
- Header pill shows total/shown counts when truncated.
