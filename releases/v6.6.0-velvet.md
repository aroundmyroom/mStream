## fix(radio): ICY stream metadata reliability

### Windows-1252 encoding fix
Some stations double-encode their stream titles ‚Äî they take the UTF-8 bytes of a character (e.g. `√â` = `C3 89`), treat each byte as a Windows-1252 character, then re-encode as UTF-8, producing garbled output like `√É‚Ä∞`. Fixed by detecting the Windows-1252 special-character fingerprint (bytes 0x80‚Äì0x9F: ‚Ç¨, ‚Ä∞, ‚Äπ, ‚Äî, ‚Ñ¢ etc.), reverse-mapping each char to its CP1252 byte, and UTF-8-decoding the result. Only accepted when the result is shorter and contains no replacement characters.

### Empty ICY tag recovery (advertisements)
When a station sends an empty `StreamTitle` (e.g. during an ad break), the now-playing display was hidden and `_radioNpLastText` was not reset ‚Äî so when the next real song came in it wasn't treated as a change and the DB badge never re-fired. Fixed: `_radioNpLastText` is reset to `null` on empty ICY, and an 8-second retry timer is scheduled so the display recovers as soon as the ad ends rather than waiting the full 30-second poll interval.

### Pause/unpause blank title
When pausing and resuming a radio stream, the now-playing title was cleared and not repopulated until the next poll cycle. Fixed: `_onAudioPlay` now immediately triggers `_pollRadioNowPlaying` for radio tracks on resume.

## fix(radio): automatic stream recovery after server restart

When the server restarts mid-stream the proxy connection drops. The existing error handler tried each fallback link once then gave up permanently.

Two fixes:
- When all fallback links are exhausted, reset to link 0 and retry automatically after 10 seconds (toast: "retrying in 10s‚Ä¶")
- In `Player.toggle()`, if the audio element is in an error state on a radio track, reload the src from scratch before calling `play()` ‚Äî so the play button always works as a manual reconnect

## feat(admin): exclude folder from music index

New `excluded` vpath type in the Directories admin. Mark any sub-folder as **"Excluded from index"** to prevent its files from ever being scanned into the music library. Any already-indexed entries under the excluded path are automatically purged the next time the parent folder is scanned.

**How it works:**
- The scanner child process skips the excluded directory entirely during traversal ‚Äî no files submitted
- `finish-scan` purges any pre-existing DB rows matching the excluded path prefix (`filepath LIKE 'prefix/%'`) after stale-file cleanup
- Changing the type to `excluded` in Admin immediately triggers a rescan of the parent folder to purge existing entries

**Admin UI:**
- Grey "üö´ Excluded from index" badge in the Directories list
- "Excluded from index" checkbox in both the Add Directory and Edit forms
- Selecting "Excluded from index" clears and disables all other type options (Audiobooks, Radio Recordings, YouTube Downloads) and unchecks "Give access to all users"
- Selecting any other type clears the Excluded checkbox

The folder remains physically accessible for browsing via the parent vpath's file explorer ‚Äî it is only excluded from the DB index.
