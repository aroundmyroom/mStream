# v6.13.2-velvet — April 2026 — Fanart Hero, Audio Output & Enrichment Fixes

## Overview

v6.13.2 brings a visual upgrade to the Artist profile page (fanart hero banner + metadata chips), completes the audio output device selector (Bluetooth disconnect handling, always-visible section), fixes four bugs in the artist image enrichment pipeline, and cleans up documentation branding throughout.

---

## New: Artist profile — fanart hero banner

When TheAudioDB provides a fanart image for an artist, the profile page now opens with a **full-width hero banner** instead of the plain header.

- The banner uses the stored fanart image at a 16:5 aspect ratio with `object-position: center top` to favour the most interesting part of wide promotional images
- A dark gradient overlay at the bottom of the banner keeps the artist name and metadata chips legible at all contrast levels
- The artist name and metadata chips (genre, country, formed year) are rendered on top of the banner when fanart is present, and fall back to the standard header layout when no fanart exists
- Responsive — the aspect ratio and padding adjust for narrow screens

---

## New: Artist profile — metadata chips

Genre, country, and formed year are now shown as small chips below the artist name — both overlaid on the fanart hero and in the fallback plain header.

These values come from TheAudioDB and are stored in the artist index after enrichment.

---

## Improved: Audio output device selector

The audio output section in **Settings** is now fully wired and always visible.

### Always visible

Previously the section was hidden when only one audio output was detected. It now stays visible at all times (unless the browser does not support `setSinkId` at all). When only one output is available, the selector is disabled with the hint: *"Connect a Bluetooth speaker or second audio device to enable."* This makes it obvious the feature exists and what to do.

### Applied consistently

`_applyOutputDevice()` is now called:
- When a crossfade (second audio) element is created
- When the main and crossfade elements are swapped during a crossfade handoff
- During the boot sequence, after queue restore

This means the selected device is honoured for all playback paths, not just the primary audio element.

### Bluetooth disconnect handling

When a Bluetooth speaker or USB audio device is disconnected, mStream Velvet now:
1. Detects the removal via the `devicechange` event
2. Resets the stored device preference to the browser default
3. Calls `setSinkId('')` on both the main and crossfade audio elements immediately so playback falls back to the default output without interruption
4. Re-renders the Settings page (if open) so the selector reflects the new state

## Fix: Artist image enrichment queue — 4 bugs

The enrichment queue ("Queue All" in Admin → Artists → Enrich) had several bugs that combined to prevent large libraries from being processed efficiently.

### 1 — Queue capacity too small (2,000 → 50,000)
`HYDRATE_QUEUE_LIMIT` was 2,000. Libraries with more than 2,000 artists without images silently stopped queueing at the cap. Raised to 50,000.

### 2 — Wrong artists returned by `getArtistsNeedingFetch`
The DB query returned all artists with `last_fetched IS NULL` — including artists that already had an image set. This wasted queue slots on artists that did not need a fetch. The query now filters to `image_file IS NULL AND last_fetched IS NULL`, ordered by `song_count DESC` so the most-played artists are enriched first.

### 3 — `seedHydrationFromMissing` re-capped at 2,000
Even after raising the queue limit, the seed function internally capped the DB query at 2,000. This cap has been removed; the seed now passes the caller-specified limit straight through.

### 4 — Rescan wiped enriched artist data
`artist-rebuild-worker.mjs` (which runs after every scan) did not include `fanart_file`, `mbid`, `genre`, `country`, or `formed_year` in its SELECT or INSERT. Every library rescan silently erased these five fields for every artist. The worker now preserves all enriched metadata across rebuilds.

## New: Admin artist enrichment — throughput display

The **Enrichment** stats panel in Admin → Artists now shows a live **throughput per minute** counter (e.g. `Throughput 12.4 /min`). This helps gauge how fast the queue is processing and whether any rate-limiting is in effect.

---

## Fix: Artist Library — back navigation restores letter

Navigating **Back** from an artist profile to the letter browse view now automatically re-expands the previously selected letter. Previously the letter list was shown collapsed and the user had to click the same letter again.

---

## Fix: YouTube download — prefer release year over upload date

The `/api/v1/ytdl/info` endpoint now uses `release_year` → `release_date` → `upload_date` in that order when populating the year field. Previously it always used `upload_date`, which for official label uploads is the video publish date — often years after the original release.

- What triggers a scan
