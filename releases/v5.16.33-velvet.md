# v5.16.33-velvet — April 2026

## Image processing: jimp → sharp

Album art compression is now handled by **sharp** (libvips) instead of jimp.

### Why the change

jimp is a pure-JavaScript image decoder. It had two recurring problems:

- **FLAC/WAV embedded art failures** — jimp occasionally failed to decode embedded images extracted from FLAC and WAV containers, producing corrupted thumbnails or falling back to the raw uncompressed file
- **Memory ceiling** — jimp decodes images to raw RGBA in JS memory; a 3000×3000 cover required ~36 MB just to produce a 256 px thumbnail, which triggered jimp's internal `maxMemoryUsageInMB` limit and caused compression to silently bail out for large embedded art

### What sharp does differently

sharp delegates all decoding and resizing to **libvips**, a C library that:
- Handles all common embedded art formats (JPEG, PNG, WebP, TIFF, AVIF) including those found inside FLAC and WAV containers
- Processes images in a streaming pipeline — memory usage is proportional to a single tile, not the entire decoded bitmap
- Is significantly faster (~5–10× for typical cover sizes)

### Files changed

| File | Change |
|------|--------|
| `src/db/scanner.mjs` | `compressAlbumArt()` rewritten with sharp; removed 8 MB bail-out that was no longer needed |
| `src/db/image-compress-script.js` | `Jimp.read()` replaced with `sharp(filepath).resize()` |
| `src/api/discogs.js` | Dynamic `import('jimp')` → dynamic `import('sharp')` — covers both Discogs and Deezer art embedding (both use the same code path) |
| `package.json` | `"jimp"` removed, `"sharp": "^0.34.5"` added |

## Bug fixes & improvements also in this release

### fix: Last.fm similar-artists and scrobbling broken (HTTP → HTTPS)
- `src/state/lastfm.js` used plain HTTP for all Last.fm API calls; Last.fm now requires HTTPS
- Auto-DJ similar-artists always fell back to random; scrobbling POST was also silently failing
- Switched to `https` module throughout; `sendGet`/`sendPost` now surface network errors; `GET /api/v1/lastfm/similar-artists` guards against null response

### fix: "Go to Player" in admin panel fails when player tab is closed
- `window.open('', 'mStream-Velvet')` returned `null` or the current window when the tab was closed
- Fixed: detects null/self and opens a new named tab; admin tab always stays open

### fix: --t3 text color too dark in Velvet and Dark themes
- Velvet `--t3`: `#6070a0` → `#7e8ec0`; Dark `--t3`: `#4a4a5a` → `#707082`
- Applied in `webapp/style.css` and `webapp/admin/index.css`

### fix: ffmpeg re-downloaded on every restart for BtbN snapshot builds
- `_getFfmpegVersion()` regex didn't match `N-<hash>` snapshot version strings → `major: 0` → fresh download every boot
- Snapshot builds now recognised and treated as `major: 99` (always valid)

### SQLite scan performance tuning
- `TX_BATCH_SIZE` raised 50 → 500 (fewer WAL fsyncs during scan)
- One-time page size migration 4 KB → 8 KB on first boot (shallower B-trees, faster queries on large libraries)

### fix: Discord URL in startup banner
- Banner now prints the correct Velvet community URL (`discord.gg/KfsTCYrTkS`) instead of the upstream server link

### docs: Rust parser compatibility analysis
- Added `docs/rust-parser-compatibility.md` — documents what upstream's experimental Rust scanner is missing and a 10-step adoption checklist
