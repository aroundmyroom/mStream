# Internationalisation (i18n)

mStream Velvet ships a full runtime i18n system for both player and admin.
Language switching is instant and does not require a page reload.

---

## Engine

Core file: `webapp/assets/js/i18n.js`

It exposes a global `I18N` object plus `window.t` alias.

| API | Description |
|-----|-------------|
| `I18N.t(key, params?)` | Translate key with fallback to English. Supports `{{param}}` interpolation and plural objects. |
| `I18N.loadLanguage(lang?)` | Load a locale (or auto-detect from `localStorage['mstream-lang']` / browser language). |
| `I18N.getLanguage()` | Return current language code. |
| `I18N.onChange(fn)` | Subscribe to language changes (returns unsubscribe). |
| `I18N.ready` | Promise that resolves after initial language load. |
| `I18N.translatePage()` | Apply translations to `[data-i18n]` DOM nodes. |
| `I18N.listLanguages()` | Return supported language metadata for UI pickers. |

Language preference is stored in `localStorage['mstream-lang']`.

---

## Safety and sync behavior

- Locale JSON is validated before activation.
- Invalid or unloadable locale files are rejected.
- On failure, UI stays on the previous valid language and shows a visible error.
- Open tabs synchronize language changes through `storage` events on `mstream-lang`.

---

## Language availability control

- Public endpoint `GET /api/v1/languages/enabled` exposes enabled language codes.
- Admin endpoints:
  - `GET /api/v1/admin/languages/config`
  - `POST /api/v1/admin/languages/config`
- Player and admin pickers only render enabled languages.
- English (`en`) is always kept enabled as fallback baseline.

---

## Locale files

All locale dictionaries live in `webapp/locales/` and must stay key-synchronized:

- `en.json` (authoritative source)
- `nl.json` (Dutch translation)
- `de.json`, `fr.json`, `es.json`, `it.json`, `pt.json`, `pl.json`, `ru.json`, `zh.json`, `ja.json`, `ko.json` (English placeholders where untranslated)

---

## UI behavior

- Player and admin use compact flag-button language pickers.
- Pickers show real country flag icons and an emoji fallback when image loading fails.
- Active language is highlighted and updates reactively.
- Static markup updates through `data-i18n` scanning after each successful language load.
- Player screens rendered from JavaScript template strings must subscribe to `I18N.onChange(...)` and rebuild the active view so open panels switch language immediately without navigation.
- The player now re-renders the current dynamic view on language change for top-level screens plus stateful views such as File Explorer, Search, saved playlists, smart-playlist results, Auto-DJ, Playback, and podcast episode lists.

---

## Translation rules

- Never hardcode user-visible text in templates or JS.
- Use i18n keys:
  - player: `t('player.*')`
  - admin: `this.t('admin.*')`
  - static HTML: `data-i18n="..."`
- Add new keys to `en.json` and `nl.json`, then sync all remaining locales.
