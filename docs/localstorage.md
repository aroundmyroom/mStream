# mStream Velvet v2 — localStorage Key Reference

All keys are written and read exclusively by `webapp/v2/app.js`.

---

## Scoping rules

The app operates in two modes:

| Mode | `S.username` | Key suffix |
|---|---|---|
| **No-auth** (server has no users configured) | `''` (empty string) | `_` (trailing underscore, e.g. `ms2_vol_`) |
| **Authenticated** | `'alice'` | `_alice` (e.g. `ms2_vol_alice`) |

### Helpers

```js
// For user-preference keys (audio, UI, playback settings):
function _uKey(k) { return `ms2_${k}_${S.username || ''}`; }

// For Auto-DJ keys:
function _djKey(k) { return `ms2_dj_${k}_${S.username || ''}`; }

// For the play queue:
function _queueKey() { return `ms2_queue_${S.username}`; }
```

Keys generated by `_uKey` and `_djKey` include the username suffix even when the server runs with no auth (`username = ''`), so settings from a no-auth session are never clobbered when a user later logs in.  
`_queueKey` requires a non-empty username; the queue is not persisted for no-auth sessions.

---

## Global keys (not user-scoped)

These identify the session itself or are set outside `app.js`:

| Key | Type | Description |
|---|---|---|
| `ms2_server_id` | UUID string | Server instance identity — set on first visit; mismatch triggers `localStorage.clear()` + reload |
| `ms2_token` | string | JWT access token — cleared on logout |
| `ms2_user` | string | Username matching the token payload |
| `ms2_logged_out` | `'1'` / absent | Set on explicit logout; prevents cookie-auth re-entry |
| `ms2_show_classic` | `'1'` / absent | Written by the admin panel; shows classic-UI links |

## User-scoped preference keys

Written by `_uKey(k)` → resolved to `ms2_<k>_<username>`.

### Playback / audio

| Suffix (`k`) | Full example key | Type | Default | Description |
|---|---|---|---|---|
| `vol` | `ms2_vol_alice` | `0`–`100` | `80` | Volume slider position |
| `balance` | `ms2_balance_alice` | `-1.0`–`1.0` | `0` | Stereo balance (panner node) |
| `crossfade` | `ms2_crossfade_alice` | integer seconds | `0` | Crossfade overlap duration; `0` = off |
| `sleep_ends` | `ms2_sleep_ends_alice` | ms timestamp or `'-1'` | absent | Sleep timer end time; `'-1'` = end of current song |

### Transcoding

| Suffix (`k`) | Type | Description |
|---|---|---|
| `trans` | `'1'` / absent | Transcoding enabled |
| `trans_codec` | string | Selected codec (e.g. `'opus'`) |
| `trans_bitrate` | string | Selected bitrate (e.g. `'128k'`) |
| `trans_algo` | string | Selected algorithm |

### Auto-DJ (written by `_djKey`)

| Suffix (`k`) | Full example key | Type | Description |
|---|---|---|---|
| `vpaths` | `ms2_dj_vpaths_alice` | JSON array | Selected source library paths |
| `min_rating` | `ms2_dj_min_rating_alice` | integer `0`–`10` | Minimum star-rating filter (`0` = any) |
| `ignore` | `ms2_dj_ignore_alice` | JSON array | Recent-play ignore list — cleared when sources change |

### Queue

| Key | Type | Description |
|---|---|---|
| `ms2_queue_<username>` | JSON object | Saved queue + playback index — requires a non-empty username |

### UI / visualiser

| Suffix (`k`) | Type | Default | Description |
|---|---|---|---|
| `autodj` | `'1'` / absent | absent | Whether Auto-DJ was active when the page was closed |
| `theme` | `'light'` \| `'dark'` | absent (follows OS) | Explicitly chosen colour scheme |
| `eq` | JSON array (8 floats) | `[0,0,0,0,0,0,0,0]` | 8-band EQ gain values in dB |
| `eq_on` | `'true'` / `'false'` | `'true'` | EQ enabled state |
| `spec_style` | integer `0`–`6` | `0` | Active spectrum visualiser mode |
| `vu_mode` | `'spec'` \| `'needle'` \| `'ppm'` | `'spec'` | VU meter display mode |
| `ppm_bright` | float `0.0`–`1.0` | `0.38` | PPM canvas brightness |
| `ref` | float `-20`–`-10` | `-13` | VU reference level in dBFS |
| `gapless` | `'1'` / absent | absent | Gapless playback enabled |
| `dynColor` | `'1'` / absent | absent | Tint UI with dominant colour sampled from current album art |

### Home view *(GitHub Copilot, 2026-03-27)*

These keys are **not** generated by `_uKey` — they use their own pattern `ms2_home_<type>_<username>`.

| Key | Type | Description |
|---|---|---|
| `ms2_home_order_<username>` | JSON array of strings | Saved shelf order for the Home view. Values are shelf IDs: `'radio'`, `'podcasts'`, `'playlists'`, `'recent'`, `'most'`. Populated by drag-to-reorder; missing IDs are appended in default order on next load. |
| `ms2_home_hidden_<username>` | JSON array of strings | IDs of cards the user has deselected in Customize mode. Format: `'rs:<id>'` (radio station), `'pf:<id>'` (podcast), `'ic:vp:<name>'` (folder), `'ic:pl:<name>'` (playlist). Absent = all cards visible. |

## Lifecycle

| Event | Action |
|---|---|
| **Page load (before login)** | IIFE fetches `/api/v1/ping/public`; compares `ms2_server_id` to server `instanceId`. Mismatch → `localStorage.clear()` + `location.reload()`. Match or first visit → sets `ms2_server_id` if absent. |
| Login (`tryLogin`) | `ms2_token` + `ms2_user` written; `ms2_logged_out` removed |
| Session restore (`checkSession` token path) | `ms2_user` read; DJ + queue keys resolved |
| Session restore (cookie/no-auth path) | `ms2_user` read (may be empty); all `_uKey` → `ms2_*_` |
| `showApp()` | Queue, Auto-DJ ignore list, sleep timer, and Auto-DJ toggle restored |
| Logout | `ms2_token` + `ms2_user` removed; `ms2_logged_out` = `'1'`; cookie expired |

## Multi-user behaviour on a shared browser

Because every user-preference key is suffixed with the username, two users (`alice` and `bob`) on the same browser maintain completely independent:
- volume, balance, EQ, crossfade
- Auto-DJ sources, rating filter, and play-history
- theme, visualiser mode, spectrum style, transcoding settings
- sleep timer and Auto-DJ on/off state

A no-auth server still gets its own isolated slot (suffix `_`).

---

## Multi-server isolation *(v6.5.2-velvet)*

localStorage is scoped by the browser to the page origin (protocol + hostname + port). Without additional protection, two mStream Velvet servers behind the same origin — or a reinstall of the same server — would share all `ms2_*` keys, including `ms2_token` and `ms2_user`.

**The `ms2_server_id` guard solves this**: on every page load, before any settings or credentials are read, the app fetches `GET /api/v1/ping/public` (no auth) and compares the returned `instanceId` to the stored `ms2_server_id`. A mismatch means a different or reinstalled server is now at this origin — `localStorage.clear()` is called immediately and the page reloads cleanly.

The `instanceId` is generated once by the server (`crypto.randomUUID()`) and persisted in `save/db/default.json`. It survives server restarts but is regenerated on a clean install.
